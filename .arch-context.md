## 9. Utilization Endpoint (Phase 38)

### Backend Controller
`backend/src/main/java/io/b2mash/b2b/b2bstrawman/capacity/UtilizationController.java`

Two endpoints:
- `GET /api/utilization/team?weekStart=YYYY-MM-DD&weekEnd=YYYY-MM-DD`
- `GET /api/utilization/members/{memberId}?weekStart=...&weekEnd=...`

Both require `weekStart` to be a Monday (enforced in controller).

### DTO (`backend/.../capacity/dto/UtilizationDtos.java`)

`TeamUtilizationResponse` contains:
- `members: List<MemberUtilizationSummary>` — per-member summary: `memberId`, `memberName`, `weeklyCapacity`, `totalPlannedHours`, `totalActualHours`, `totalBillableHours`, `avgPlannedUtilizationPct`, `avgActualUtilizationPct`, `avgBillableUtilizationPct`, `overAllocatedWeeks`, `weeks: List<WeekUtilization>`
- `teamAverages: TeamAverages` — org-wide `avgPlanned/Actual/BillableUtilizationPct`

**One-line shape summary**: *Returns per-member planned-vs-actual-vs-billable hours and utilization percentages broken down by week, plus team-wide averages — so a dashboard widget can render a single "team billable utilization" KPI by reading `teamAverages.avgBillableUtilizationPct`.*

### Frontend Consumers
- API client: `frontend/lib/api/capacity.ts`
- Full utilization page: `frontend/app/(app)/org/[slug]/resources/utilization/page.tsx`
- Chart: `frontend/components/capacity/utilization-chart.tsx`
- Table: `frontend/components/capacity/utilization-table.tsx`

- **No `@Filter` / `@FilterDef`** — Phase 13 removed all row-level filtering. Tenant isolation is schema-per-tenant via `ScopedValue` (`RequestScopes.TENANT_ID`) and Hibernate `search_path`.
- **Pack content has no tenancy concerns** — packs are classpath resources, not per-tenant data. `PackInstallService.internalInstall()` is called per-tenant during provisioning; the pack files themselves are org-agnostic.
- **Backend controller discipline** — controllers must be one-line delegations; no business logic, no repositories injected. (See `backend/CLAUDE.md`.)
