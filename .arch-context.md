### 7.2 `AccountingSyncEntry` (sync queue entity / state machine) — FULL structure
> The canonical "sync queue row + state machine" to mirror for calendar/email sync. Enums: `SyncDirection {PUSH, PULL}`, `SyncEntityType {INVOICE, CUSTOMER, PAYMENT_PULL}`, `SyncTrigger {EVENT, MANUAL_RETRY, FORCE_RESYNC}`, `SyncState {PENDING, IN_FLIGHT, COMPLETED, FAILED_RETRYING, DEAD_LETTER, BLOCKED_TRUST_BOUNDARY, RECONCILE_DRIFT}`.
```java
@Entity @Table(name = "accounting_sync_entry")
public class AccountingSyncEntry {
  @Id @GeneratedValue(strategy=GenerationType.UUID) private UUID id;
  @Enumerated(EnumType.STRING) @Column(name="entity_type",nullable=false,length=20) private SyncEntityType entityType;
  @Column(name="entity_id",nullable=false) private UUID entityId;
  @Column(name="provider_id",nullable=false,length=20) private String providerId;
  @Enumerated(EnumType.STRING) @Column(name="direction",nullable=false,length=10) private SyncDirection direction;
  @Enumerated(EnumType.STRING) @Column(name="state",nullable=false,length=30) private SyncState state;
  @Column(name="attempt_count",nullable=false) private int attemptCount;
  @Column(name="next_attempt_at") private Instant nextAttemptAt;
  @Column(name="last_error_code",length=50) private String lastErrorCode;
  @Column(name="last_error_detail",columnDefinition="TEXT") private String lastErrorDetail;
  @Column(name="external_reference",length=100) private String externalReference;
  @Column(name="external_id",length=100) private String externalId;
  @Enumerated(EnumType.STRING) @Column(name="\"trigger\"",nullable=false,length=30) private SyncTrigger trigger; // quoted reserved word
  @Column(name="created_at",nullable=false,updatable=false) private Instant createdAt;
  @Column(name="updated_at",nullable=false) private Instant updatedAt;
  @Column(name="completed_at") private Instant completedAt;
  protected AccountingSyncEntry() {}
  public AccountingSyncEntry(SyncEntityType, UUID entityId, String providerId, SyncDirection, SyncTrigger, String externalReference){...state=PENDING; attemptCount=0; nextAttemptAt=Instant.now();...}
  @PrePersist/@PreUpdate set timestamps.
  // State transitions:
  markInFlight()                 // -> IN_FLIGHT, nextAttemptAt=null
  markCompleted(externalId)      // -> COMPLETED, sets externalId+completedAt, clears errors
  markFailedRetrying(code,detail,next) // -> FAILED_RETRYING, attemptCount++
  markRateLimited(next)          // -> PENDING, "RATE_LIMIT", does NOT consume attempt budget
  markDeadLetter(code,detail)    // -> DEAD_LETTER
  markBlockedTrustBoundary(reason)// -> BLOCKED_TRUST_BOUNDARY (permanent)
  markReconcileDrift(code,detail)// -> RECONCILE_DRIFT
  resetForRetry()                // DEAD_LETTER -> PENDING, attemptCount=0, trigger=MANUAL_RETRY (throws if not DEAD_LETTER)
}
```

### 7.3 `AccountingSyncWorker` (scheduled multi-tenant drain) — FULL pattern
```java
@Service
public class AccountingSyncWorker {
  private static final int BATCH_SIZE = 25, MAX_ATTEMPTS = 5;
  private static final Duration RATE_LIMIT_DELAY = Duration.ofSeconds(60);
  private static final Duration[] BACKOFF_SCHEDULE = {1m, 5m, 15m, 1h, 6h};
  // ctor injects: AccountingSyncEntryRepository, IntegrationRegistry, AccountingSyncService,
  //               ApplicationEventPublisher, TransactionTemplate, JobEnqueuer

  @SchedulerLock(name="accounting_sync_drain_pending_entries", lockAtLeastFor="15s")
  @Scheduled(fixedDelay = 30_000)
  public void drainPendingEntries() { jobEnqueuer.fanOutToAllTenants("accounting_sync_drain", null); }

  int drainForTenant() {  // called per-tenant by AccountingSyncDrainHandler inside tenant scope
    var entries = syncEntryRepository.findDrainableEntries(Instant.now(), PageRequest.of(0, BATCH_SIZE));
    int processed=0;
    for (var entry : entries) {
      Boolean ok = transactionTemplate.execute(tx -> processEntry(entry));
      if (Boolean.TRUE.equals(ok)) processed++;
      else if (ok == null) break;   // rate-limited -> stop this tenant
    }
    return processed;
  }

  private Boolean processEntry(AccountingSyncEntry entry) {
    entry.markInFlight(); save;
    if (!syncService.checkTrustBoundary(entry).allowed()) { entry.markBlockedTrustBoundary(reason); save; return true; }
    var provider = integrationRegistry.resolve(IntegrationDomain.ACCOUNTING, AccountingProvider.class);
    var result = dispatchSync(provider, entry);                  // switch on entityType
    if (result.success()) { entry.markCompleted(externalId); save; publish XeroSyncEntryCompletedEvent; return true; }
    // classify error: rate-limit -> markRateLimited + return null; validation -> dead-letter;
    // transient -> backoff retry or dead-letter at MAX_ATTEMPTS (publish XeroSyncEntryDeadLetteredEvent)
    // catch(Exception) -> persist FAILED_RETRYING or DEAD_LETTER; return true
  }
}
```
> Rate-limit detection / validation detection are string-based on the provider error message. `computeNextAttempt(n)` indexes BACKOFF_SCHEDULE (clamped).

### 7.4 `XeroOAuthService` (OAuth2 authorization-code + PKCE token lifecycle) — FULL pattern
> **The model for Google/Microsoft calendar + Gmail/Graph OAuth.** Injects `XeroProperties, SecretStore, OrgIntegrationRepository, AccountingXeroConnectionRepository, AccountingTaxCodeMappingService, AuditService, ApplicationEventPublisher, XeroApiClient, @Qualifier("xeroTokenClient") RestClient, ObjectMapper`.
- `initiateConnect(memberId)` → `XeroConnectResult(authorizationUrl, state)`: generate PKCE verifier (32 random bytes, base64-url-no-pad) + SHA-256 challenge; `state=UUID`; `secretStore.store("xero:oauth:state:"+state, codeVerifier)`; build authorize URL (`response_type=code&client_id&redirect_uri&scope&state&code_challenge&code_challenge_method=S256`).
- `handleCallback(code, state, memberId)` `@Transactional` → `XeroCallbackResult(connectionId, xeroOrgName)`: validate+consume state from SecretStore (delete after); exchange code→tokens (form-urlencoded POST); fetch remote connections for tenant id/name; **upsert `OrgIntegration`** (`findByDomain(ACCOUNTING).orElseGet(new OrgIntegration(ACCOUNTING,"xero"))`, `updateProvider`, `enable`, save); create `AccountingXeroConnection`; **store tokens in SecretStore** keyed `integrationId+":xero:access"` / `":xero:refresh"`; pre-seed tax mappings; publish `XeroConnectionEstablishedEvent`; `auditService.log(AuditEventBuilder.builder().eventType("integration.xero.connected")...)`.
- `refreshAccessToken(connectionId)` — **intentionally NOT `@Transactional`** (so failure-state writes survive a re-throw): refresh tokens, store new ones; on 3 consecutive failures `connection.markRefreshFailed()` + audit `"integration.xero.refresh_failed"`.
- `disconnect(memberId)` `@Transactional`: best-effort token revocation, delete secrets, `connection.markRevoked()`, disable OrgIntegration, publish `XeroConnectionRevokedEvent`, audit `"integration.xero.disconnected"`.
- Settings persisted as JSON in `OrgIntegration.configJson` (parse/serialize via Jackson — **`tools.jackson.*`**, the Boot 4 package, NOT `com.fasterxml.jackson`). `XeroSyncSettings(paymentPollIntervalMinutes, pushTrigger, autoSyncEnabled)`.
- Token HTTP via dedicated `RestClient` with `.onStatus(HttpStatusCode::isError, ...)` error mapping; helpers `exchangeCodeForTokens`, `refreshTokens`, `revokeToken` (all form-urlencoded with client_id+client_secret).

### 7.8 `IntegrationRegistry` — FULL (key behaviours)
> Scans all `@IntegrationAdapter` beans at startup into `domain → slug → bean` (fails fast on duplicate domain+slug). 60s Caffeine cache of per-tenant `OrgIntegration` config keyed `"tenantSchema:DOMAIN"`.
- `resolve(domain, portInterface)` — **must run inside a tenant-scoped context** (`RequestScopes.TENANT_ID.isBound()`); returns the configured-and-enabled adapter, else the domain's default-slug adapter; throws if no default registered or the adapter doesn't implement the port.
- `resolveBySlug(domain, slug, portInterface)` — bypasses tenant config; **used by webhook controllers before tenant context exists.** Caller MUST allowlist `slug` (never pass a raw header/body slug). Throws `IllegalArgumentException` for unknown slug (do not leak verbatim).
- `availableProviders(domain)`; `evict(tenantSchema, domain)` (call on config change).

### 8.3 `EmailWebhookService` — FULL pattern (closest precedent for an inbound webhook pipeline)
> Processes SendGrid delivery-event webhooks. Load-bearing patterns:
- **Tenant resolution from webhook payload, not JWT**: reads `tenantSchema` from `event.unique_args.tenantSchema`, validates against `Pattern "^tenant_[0-9a-f]{12}$"`, then runs work inside `RequestScopes.runForTenant(tenantSchema, null, () -> {...})`. **This is exactly how an inbound email→matter webhook binds a tenant.**
- **Signature verification fail-closed**: rejects all requests if the verification key is unset; verifies ECDSA signature (headers `X-Twilio-Email-Event-Webhook-Signature` + `-Timestamp`) via SendGrid `EventWebhook`. Throws `WebhookAuthenticationException` (→401) / `WebhookPayloadException` (→400).
- Provider path-variable sanitized + length-capped before logging (anti log-injection).
- Per-event processing intentionally NOT wrapped in one transaction (idempotent; partial success preferred).
- Jackson is `tools.jackson.databind.ObjectMapper`.

### 8.4 `EmailWebhookController` — FULL
```java
@RestController @RequestMapping("/api/webhooks/email")
public class EmailWebhookController {
  private final EmailWebhookService webhookService;
  public EmailWebhookController(EmailWebhookService s){ this.webhookService=s; }
  @PostMapping("/{provider}")
  public ResponseEntity<Void> handleWebhook(
      @PathVariable String provider, @RequestBody String payload,
      @RequestHeader(value="X-Twilio-Email-Event-Webhook-Signature", required=false) String signature,
      @RequestHeader(value="X-Twilio-Email-Event-Webhook-Timestamp", required=false) String timestamp) {
    try { webhookService.processWebhook(provider, payload, signature, timestamp); return ResponseEntity.ok().build(); }
    catch (WebhookAuthenticationException e){ return ResponseEntity.status(401).build(); }
    catch (WebhookPayloadException e){ return ResponseEntity.badRequest().build(); }
  }
}
```
> Webhook controllers live under `/api/webhooks/{provider}`, are public (no JWT), and self-authenticate via signature. An inbound-email webhook (SendGrid/Mailgun Inbound Parse, or a poll alternative) follows this shape, e.g. `/api/webhooks/inbound-email/{provider}`.
> **Other `/api/webhooks/*` precedents**: `integration/payment/PaymentWebhookController` + `PaymentWebhookService`. NOTE `integration/accounting/sync/AccountingSyncController` is `@RequestMapping("/api/integrations/sync")` (JWT-authed sync status/retry, NOT a webhook).

## 16. Cross-Cutting Conventions (for the architect)

- **Tenant isolation = schema-per-tenant only.** No `tenant_id` columns, no `@Filter`, no RLS. Tenant tables under `db/migration/tenant/`; global (cross-tenant) under `db/migration/global/`.
- **Jackson is `tools.jackson.*`** (Spring Boot 4), not `com.fasterxml.jackson.*`.
- **OAuth integrations**: mirror `XeroOAuthService` (PKCE, state in SecretStore, tokens in SecretStore, OrgIntegration upsert, connection entity, domain events, audit; refresh method deliberately non-transactional).
- **Sync work**: mirror `AccountingSyncEntry` (state machine) + `AccountingSyncWorker` (`@SchedulerLock` + `@Scheduled` + `jobEnqueuer.fanOutToAllTenants` + per-tenant `drainForTenant()` with backoff/dead-letter) + `AccountingSyncEventListener` (`@TransactionalEventListener(AFTER_COMMIT)` enqueues entries; AFTER_COMMIT is shard-aware).
- **Inbound webhooks**: mirror `EmailWebhookController`/`EmailWebhookService` (`/api/webhooks/{provider}`, public, signature-verified fail-closed, tenant bound from validated `tenant_<12hex>` payload via `RequestScopes.runForTenant`, `IntegrationRegistry.resolveBySlug` with allowlisted slug).
- **New integration domains** need a registered NoOp default adapter or the registry throws on resolve.
- **Controllers are thin HTTP adapters** (one service call, return ResponseEntity, `@RequiresCapability` for authz — see backend/CLAUDE.md). Several existing controllers (incl. the older Calendar/Reporting ones using EntityManager-in-service) violate conventions; do not copy them.
- **Quality gate**: full `./mvnw verify` (backend) + `pnpm lint && pnpm build && pnpm test` (frontend) before merge. No Testcontainers in tests (embedded Postgres + in-memory storage). GreenMail SMTP singleton on port 13025 for email tests.
- **Frontend**: RSC by default; no Server→Client function/component-ref props; SWR for client fetching; slate palette + teal accent (Signal Deck); unified `radix-ui` package; Next.js 16 async `params` (await).
