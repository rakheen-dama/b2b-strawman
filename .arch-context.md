## 10. Key Patterns Summary (for the architect)

### Detail Page Pattern
- Server Component page fetches all data with `api.get<T>()` in parallel `Promise.all()`
- Each fetch wrapped in try/catch for non-fatal failures
- Capability-gated data (rates, financials) only fetched when `canManage`
- Module-gated tabs (court dates, trust, disbursements) conditionally rendered
- Custom fields, tags, saved views fetched alongside core data
- Tabbed UI via `ProjectTabs` client component with Radix Tabs + Motion indicator

### Tab Addition Pattern
1. Add `ReactNode` prop to `ProjectTabsProps` (optional for new tabs)
2. Add TabId to the union type
3. Add to `buildBaseTabs()` array
4. Add to `validTabIds` set
5. Add module-gating logic in the component body
6. Add `TabsPrimitive.Content` block
7. Pass panel content from the page component

### Data Fetching Convention
- `api.get<T>()` for typed backend calls
- `handleApiError()` for fatal errors (project not found)
- Silent catch for non-fatal (features degrade gracefully)
- `Promise.all()` / `Promise.allSettled()` for parallel fetches
