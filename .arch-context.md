## 7. Reference Entity Pattern -- AutomationRule

This entity demonstrates the full pattern for a tenant-scoped entity with JSONB columns, enums, and lifecycle methods.

File: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/automation/AutomationRule.java`

```java
package io.b2mash.b2b.b2bstrawman.automation;

import jakarta.persistence.Column;
import jakarta.persistence.Entity;
import jakarta.persistence.EnumType;
import jakarta.persistence.Enumerated;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.GenerationType;
import jakarta.persistence.Id;
import jakarta.persistence.Table;
import java.time.Instant;
import java.util.List;
import java.util.Map;
import java.util.UUID;
import org.hibernate.annotations.JdbcTypeCode;
import org.hibernate.type.SqlTypes;

@Entity
@Table(name = "automation_rules")
public class AutomationRule {

  @Id
  @GeneratedValue(strategy = GenerationType.UUID)
  private UUID id;

  @Column(name = "name", nullable = false, length = 200)
  private String name;

  @Column(name = "description", length = 1000)
  private String description;

  @Column(name = "enabled", nullable = false)
  private boolean enabled;

  @Enumerated(EnumType.STRING)
  @Column(name = "trigger_type", nullable = false, length = 50)
  private TriggerType triggerType;

  @JdbcTypeCode(SqlTypes.JSON)
  @Column(name = "trigger_config", columnDefinition = "jsonb", nullable = false)
  private Map<String, Object> triggerConfig;

  @JdbcTypeCode(SqlTypes.JSON)
  @Column(name = "conditions", columnDefinition = "jsonb")
  private List<Map<String, Object>> conditions;

  @Enumerated(EnumType.STRING)
  @Column(name = "source", nullable = false, length = 20)
  private RuleSource source;

  @Column(name = "template_slug", length = 100)
  private String templateSlug;

  @Column(name = "created_by", nullable = false)
  private UUID createdBy;

  @Column(name = "created_at", nullable = false, updatable = false)
  private Instant createdAt;

  @Column(name = "updated_at", nullable = false)
  private Instant updatedAt;

  protected AutomationRule() {}

  public AutomationRule(
      String name,
      String description,
      TriggerType triggerType,
      Map<String, Object> triggerConfig,
      List<Map<String, Object>> conditions,
      RuleSource source,
      String templateSlug,
      UUID createdBy) {
    this.name = name;
    this.description = description;
    this.enabled = true;
    this.triggerType = triggerType;
    this.triggerConfig = triggerConfig;
    this.conditions = conditions;
    this.source = source;
    this.templateSlug = templateSlug;
    this.createdBy = createdBy;
    this.createdAt = Instant.now();
    this.updatedAt = Instant.now();
  }

  public void toggle() {
    this.enabled = !this.enabled;
    this.updatedAt = Instant.now();
  }

  public void update(
      String name,
      String description,
      TriggerType triggerType,
      Map<String, Object> triggerConfig,
      List<Map<String, Object>> conditions) {
    this.name = name;
    this.description = description;
    this.triggerType = triggerType;
    this.triggerConfig = triggerConfig;
    this.conditions = conditions;
    this.updatedAt = Instant.now();
  }

  // ... getters omitted for brevity (standard pattern: one getter per field)
}
```

### Key patterns to follow:
- **No multitenancy boilerplate** -- schema-per-tenant handles isolation (see ADR-064)
- `@GeneratedValue(strategy = GenerationType.UUID)` for all IDs
- `protected` no-arg constructor for JPA
- `Instant` for timestamps, `LocalDate` for date-only fields
- JSONB columns use `@JdbcTypeCode(SqlTypes.JSON)` + `@Column(columnDefinition = "jsonb")`
- Enums use `@Enumerated(EnumType.STRING)`
- Business logic methods (toggle, update) on the entity -- not anemic model
- `createdAt`/`updatedAt` set in constructor, `updatedAt` updated in mutators

```java
public abstract class AbstractPackSeeder<D> {
  // Template method pattern:
  // 1. getPackResourcePattern() -- classpath glob for JSON files
  // 2. getPackDefinitionType() -- Java class to deserialize into
  // 3. getPackTypeName() -- "field", "template", "clause", etc.
  // 4. getPackId(D pack) -- extract pack identifier
  // 5. getPackVersion(D pack) -- extract version string
  // 6. isPackAlreadyApplied(OrgSettings, packId) -- idempotency check
  // 7. recordPackApplication(OrgSettings, pack) -- mark as applied
  // 8. applyPack(D pack, Resource, tenantId) -- create domain entities

  // Entry point:
  public void seedPacksForTenant(String tenantId, String orgId) {
    tenantTransactionHelper.executeInTenantTransaction(tenantId, orgId, t -> doSeedPacks(t));
  }
  // Loads JSON from classpath, checks OrgSettings for idempotency, applies each unapplied pack
}
```

## 10. Field Pack Seeder Pattern

The `applyPack` method:
1. Creates a `FieldGroup` from the pack's group definition
2. Iterates over `pack.fields()`, creating a `FieldDefinition` for each
3. Links each field to the group via `FieldGroupMember`
4. Supports: required, defaultValue, options (dropdown), validation, requiredForContexts (prerequisite contexts)

## 11. Compliance Pack Seeder Pattern

File: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/compliance/CompliancePackSeeder.java`

The `applyPack` method:
1. Creates a `ChecklistTemplate` from `pack.checklistTemplate()`
2. First pass: creates `ChecklistTemplateItem` entries (no dependencies)
3. Second pass: resolves inter-item dependencies via `dependsOnItemKey`
4. Cross-seeds `FieldDefinition` + `FieldGroup` + `FieldGroupMember` if `pack.fieldDefinitions()` is present
5. Seeds `RetentionPolicy` entries if `pack.retentionOverrides()` is present

## 12. Document Template Pack Seeder Pattern

## 13. Clause Pack Seeder Pattern

File: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/clause/ClausePackSeeder.java`

The `applyPack` method:
1. Seeds `Clause` entities from `pack.clauses()` -- skips if slug already exists
2. Clause body is stored as JSONB Map (TipTap format)
3. Uses `Clause.createSystemClause()` factory method (source=SYSTEM, active=true)
4. Creates `TemplateClause` associations from `pack.templateAssociations()` -- links clauses to document templates by packId + templateKey

## 14. Automation Rule Template Seeder Pattern

File: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/automation/template/AutomationTemplateSeeder.java`

The `applyPack` method:
1. Iterates over `pack.templates()`

5. Uses `SYSTEM_USER_ID` (`00000000-...`) as createdBy
