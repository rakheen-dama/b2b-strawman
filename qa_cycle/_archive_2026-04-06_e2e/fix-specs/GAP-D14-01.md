# Fix Spec: GAP-D14-01 — Conflict check doesn't search adverse party names

## Problem
Searching "Mokoena" in conflict check returns NO_CONFLICT despite "vs Mokoena" existing in a matter name. The conflict check only matches client names and adverse party names — it does NOT search project/matter names or descriptions.

## Root Cause (confirmed)
The `ConflictCheckService.performCheck()` (lines 111-308 in `ConflictCheckService.java`) searches:
1. Adverse parties by ID number (exact)
2. Adverse parties by registration number (exact)
3. Adverse parties by name similarity (fuzzy, `adversePartyRepository.findBySimilarName`)
4. Adverse parties by alias (fuzzy)
5. Customers by name similarity (`customerRepository.findBySimilarName`)

It does **NOT** search:
- Project/matter names
- Project/matter descriptions
- Customer email, ID numbers (only direct name match)

The test plan expects searching "Mokoena" to match against matter name "Debt Recovery - vs Mokoena (R45,000)". This requires adding project name search.

Note: By Day 75, the QA cycle shows "Mokoena" DID return a conflict — but that's because an adverse party named "T. Mokoena" was added via DB. The gap is about searching project/matter names themselves.

## Fix

**`backend/src/main/java/io/b2mash/b2b/b2bstrawman/verticals/legal/conflictcheck/ConflictCheckService.java`:**

After the customer name matching block (around line 244), add project name matching:

```java
// Step 2c: Search project names for potential matches
var projectNameMatches = projectRepository.findBySimilarName(
    request.checkedName(), FUZZY_THRESHOLD, MAX_SEARCH_RESULTS);
for (var project : projectNameMatches) {
  double score = estimateSimilarity(project.getName(), request.checkedName());
  if (score > highestScore) {
    highestScore = score;
  }
  // Find the linked customer for context
  var customerLinks = customerProjectRepository.findByProjectId(project.getId());
  UUID linkedCustomerId = customerLinks.isEmpty() ? null : customerLinks.getFirst().getCustomerId();
  String linkedCustomerName = null;
  if (linkedCustomerId != null) {
    linkedCustomerName = customerRepository.findById(linkedCustomerId)
        .map(Customer::getName).orElse(null);
  }
  conflicts.add(new ConflictDetail(
      null, null, linkedCustomerId, linkedCustomerName,
      project.getId(), project.getName(),
      "MATTER_NAME", "NAME_SIMILARITY", score,
      buildExplanation("NAME_SIMILARITY", project.getName(), request.checkedName(), score)));
}
```

**`backend/src/main/java/io/b2mash/b2b/b2bstrawman/project/ProjectRepository.java`:**
Add a `findBySimilarName` query using `pg_trgm` (similar to the existing ones in `CustomerRepository` and `AdversePartyRepository`).
