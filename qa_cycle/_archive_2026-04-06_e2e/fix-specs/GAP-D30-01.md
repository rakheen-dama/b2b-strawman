# Fix Spec: GAP-D30-01 — LSSA tariff items show "R NaN"

## Problem
All 19 LSSA tariff items across 7 sections display "R NaN" in the Add Tariff Items dialog. Users cannot add tariff line items to invoices. This is a fork-readiness blocker.

## Root Cause (confirmed)
**Field name mismatch between backend DTO and frontend type.**

The backend `TariffService.TariffItemResponse` (line 92-101 in `TariffService.java`) returns `amount` (BigDecimal in rands, e.g. `850.00`):
```java
public record TariffItemResponse(
    UUID id, UUID scheduleId, String itemNumber, String section,
    String description, BigDecimal amount, String unit, String notes, int sortOrder) {}
```

The frontend `TariffItem` type (`frontend/lib/types/legal.ts:178-188`) expects `rateInCents` (number):
```typescript
export interface TariffItem {
  rateInCents: number;
  // ...
}
```

When JSON `{"amount": 850.00}` is deserialized to `TariffItem`, `rateInCents` is `undefined`. The `formatZAR` function divides `undefined / 100` which produces `NaN`.

Additionally, the `TariffLineDialog` (line 131) sends `unitPrice: item.rateInCents / 100` which would be `NaN / 100 = NaN`.

## Fix
**Option A (recommended): Align frontend to match backend field name**

1. **`frontend/lib/types/legal.ts`** — Change `TariffItem.rateInCents` to `amount`:
   ```typescript
   export interface TariffItem {
     amount: number; // BigDecimal in rands (not cents)
     // ...
   }
   ```

2. **`frontend/components/legal/tariff-item-browser.tsx`** — Update `formatZAR` and references:
   - Change `formatZAR` to take rands not cents: `return \`R ${amount.toFixed(2)}\``
   - Change line 195: `formatZAR(item.amount)`

3. **`frontend/components/legal/tariff-line-dialog.tsx`** — Update references:
   - Line 28-29: `formatZAR` to format rands: `return \`R ${amount.toFixed(2)}\``
   - Line 117: `sum + item.amount * quantity` (not rateInCents)
   - Line 131: `unitPrice: item.amount` (already in rands, no division needed)
   - Line 216: `formatZAR(item.amount)`

4. **Update tests**:
   - `frontend/__tests__/legal/tariff-schedule.test.tsx` — Change `rateInCents` to `amount` in mock data
   - `frontend/__tests__/legal/tariff-invoice.test.tsx` — Same
