# Fix Spec: GAP-D60-02 — Customer column blank in profitability report

## Problem
The Customer/Client column in the profitability report shows "--" (dash) for all rows despite clients being linked to projects/matters.

## Root Cause (hypothesis)
The backend `ReportService.orgProfitabilityReport()` (lines 252-268 in `ReportService.java`) correctly builds a `customerNameByProject` map by looking up `CustomerProject` links. The logic is:

```java
var links = customerProjectRepository.findByProjectId(projectId);
if (!links.isEmpty()) {
  var customer = customerRepository.findById(links.getFirst().getCustomerId());
  customer.ifPresent(c -> customerNameByProject.put(projectId, c.getName()));
}
```

This should work. However, the `CustomerProject` links must exist. Two possible root causes:

1. **CustomerProject not created for API-created projects.** Projects created via API during Day 2-3 may not have `CustomerProject` links if the creation endpoint doesn't establish them. But the template instantiation flow (line 524-527 in `ProjectTemplateService.java`) DOES create `CustomerProject` links. API-created projects would need a separate `POST /api/customers/{id}/projects` call.

2. **Frontend not passing `customerName` through.** The `OrgProfitabilityResponse` includes `ProjectProfitabilitySummary` which has `customerName`. The frontend report component may not be rendering this field.

Need to check the frontend report component to confirm which rendering issue it is.

## Fix

**Step 1: Verify the data** — Check if `customerNameByProject` is populated:
```bash
curl http://localhost:8081/api/reports/org-profitability?from=2026-04-01&to=2026-04-30
```
If `customerName` is null in the response, the issue is in the backend data (missing `CustomerProject` links for API-created projects). If populated, the issue is frontend rendering.

**Step 2 (if backend data issue):** Ensure `CustomerProject` links exist for all projects. The Day 2-3 QA execution created projects via API with customer linking — verify the linking endpoint creates `CustomerProject` records.

**Step 3 (if frontend rendering issue):** Fix the profitability report component to render `customerName` from the response.
