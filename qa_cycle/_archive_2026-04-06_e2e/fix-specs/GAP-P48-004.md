# GAP-P48-004: Trust Field Pack Not Registered in Vertical Profile

## Decision: SPEC_READY

## Effort: S (10 minutes)

## Problem

The trust-specific field pack exists at `backend/src/main/resources/field-packs/accounting-za-customer-trust.json` with 6 fields (Trust Registration Number, Trust Deed Date, Trust Type, Names of Trustees, Trustee Appointment Type, Letters of Authority Date). However, it is NOT registered in the vertical profile's `packs.field` array.

Current value in `backend/src/main/resources/vertical-profiles/accounting-za.json` line 8:
```json
"field": ["accounting-za-customer", "accounting-za-project"]
```

## Fix

In `backend/src/main/resources/vertical-profiles/accounting-za.json`, change the `field` array to include the trust pack:

```json
"field": ["accounting-za-customer", "accounting-za-project", "accounting-za-customer-trust"]
```

## Files to Modify

| File | Action |
|------|--------|
| `backend/src/main/resources/vertical-profiles/accounting-za.json` | Add `"accounting-za-customer-trust"` to `packs.field` array |

## Verification

After the fix, when a new org is provisioned with the `accounting-za` profile, the trust fields should be seeded into the tenant schema. The QA script at Day 2-3 (step 2.11) creates a "Moroka Family Trust" customer — the trust-specific fields should appear when entity type is Trust.

Note: This fix only ensures the pack is seeded. GAP-P48-008 (FICA checklist does not filter by entity type) is a separate issue that controls visibility of these fields based on entity type.
