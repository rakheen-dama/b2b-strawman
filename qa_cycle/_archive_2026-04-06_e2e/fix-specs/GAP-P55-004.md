# Fix Spec: GAP-P55-004 — Court Calendar missing date range and client filters

## Problem
The Court Calendar list view has Status and Type filter dropdowns but no client/matter name search or date range (from/to) filters. The test plan expects these.

## Root Cause (confirmed)
- Client: `frontend/app/(app)/org/[slug]/court-calendar/court-calendar-client.tsx` line 149-183
  - Only `statusFilter` and `dateTypeFilter` state variables exist
  - No `customerFilter`, `projectFilter`, `dateFrom`, or `dateTo` states
- Actions: `frontend/app/(app)/org/[slug]/court-calendar/actions.ts` line 31-38
  - `CourtDateFilters` interface already supports `from`, `to`, `customerId`, `projectId` — they are just not wired to the UI
- Backend: `CourtCalendarController.java` line 40-56
  - Already accepts `dateFrom`, `dateTo`, `customerId`, `projectId` query params

The backend and action layer support all filters. Only the UI is missing the filter controls.

## Fix
1. In `court-calendar-client.tsx`, add state variables:
   ```tsx
   const [dateFrom, setDateFrom] = useState("");
   const [dateTo, setDateTo] = useState("");
   const [clientSearch, setClientSearch] = useState("");
   ```

2. Add date range inputs and a client search input to the filter bar (line 151-183):
   ```tsx
   <Input type="date" value={dateFrom} onChange={e => setDateFrom(e.target.value)} placeholder="From" />
   <Input type="date" value={dateTo} onChange={e => setDateTo(e.target.value)} placeholder="To" />
   <Input value={clientSearch} onChange={e => setClientSearch(e.target.value)} placeholder="Search client/matter..." />
   ```

3. Wire `dateFrom` and `dateTo` into the `refetchCourtDates` function's filter object:
   ```tsx
   const filters: CourtDateFilters = {};
   if (statusFilter) filters.status = statusFilter;
   if (dateTypeFilter) filters.dateType = dateTypeFilter;
   if (dateFrom) filters.from = dateFrom;
   if (dateTo) filters.to = dateTo;
   ```

4. For client search, the backend filters by `customerId` not by name search. Either:
   - Add a text search endpoint on the backend (more work), OR
   - Filter client-side on `courtDates[].customerName` (simpler for small datasets)
