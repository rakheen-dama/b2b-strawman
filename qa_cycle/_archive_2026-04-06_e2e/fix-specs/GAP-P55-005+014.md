# Fix Spec: GAP-P55-005 + GAP-P55-014 — Cosmetic display issues (enum formatting + terminology)

## Problem
Two cosmetic display issues:
1. **GAP-P55-005**: Court date type badge shows "Pre_trial" with underscore instead of "Pre-Trial". The `dateTypeLabel()` function only capitalizes the first character and lowercases the rest, which doesn't handle multi-word enum values with underscores.
2. **GAP-P55-014**: Project detail page shows "Customer: Sipho Mabena" instead of "Client:" for legal-za tenants. The sidebar correctly uses "Clients" terminology but the detail page has a hardcoded "Customer:" label.

## Root Cause (confirmed)

### GAP-P55-005 — Enum display
- `frontend/components/legal/court-date-list-view.tsx` line 46-48:
  ```tsx
  function dateTypeLabel(type: string): string {
    return type.charAt(0) + type.slice(1).toLowerCase();
  }
  ```
  This turns `"PRE_TRIAL"` into `"Pre_trial"` instead of `"Pre-Trial"`.

- Same function duplicated in `frontend/components/legal/upcoming-court-dates-widget.tsx` line 38-40.
- The detail panel in `court-calendar-client.tsx` line 293-294 uses the same pattern inline:
  ```tsx
  {selectedCourtDate.dateType.charAt(0) + selectedCourtDate.dateType.slice(1).toLowerCase()}
  ```

### GAP-P55-014 — Terminology
- `frontend/app/(app)/org/[slug]/projects/[id]/page.tsx` line 461:
  ```tsx
  Customer:{" "}
  ```
  This is a hardcoded string. It should use the terminology system: `{t("Customer")}:` via `useTerminology()` hook or `<TerminologyText template="{Customer}:" />`.
- However, this is a Server Component (`page.tsx` with no `"use client"`), so it cannot use the `useTerminology()` hook directly. The fix requires either:
  - Extracting the customer link into a client component that uses `useTerminology()`, OR
  - Using the `<TerminologyText>` client component inline

## Fix

### GAP-P55-005 — Fix dateTypeLabel
1. Replace the `dateTypeLabel` function in all locations with a proper formatter:
   ```tsx
   function dateTypeLabel(type: string): string {
     return type
       .split("_")
       .map(word => word.charAt(0).toUpperCase() + word.slice(1).toLowerCase())
       .join("-");
   }
   ```
   This turns `"PRE_TRIAL"` into `"Pre-Trial"`, `"CASE_MANAGEMENT"` into `"Case-Management"`, etc.

2. Apply in:
   - `frontend/components/legal/court-date-list-view.tsx` line 46-48
   - `frontend/components/legal/upcoming-court-dates-widget.tsx` line 38-40
   - `frontend/app/(app)/org/[slug]/court-calendar/court-calendar-client.tsx` line 293-294

### GAP-P55-014 — Fix terminology
3. In `frontend/app/(app)/org/[slug]/projects/[id]/page.tsx` line 461, replace the hardcoded "Customer:" with a `<TerminologyText>` component:
   ```tsx
   <TerminologyText template="{Customer}:" />
   ```
   Since `TerminologyText` is a client component, it can be used inline in a Server Component as a leaf node (it's imported and rendered as JSX).
