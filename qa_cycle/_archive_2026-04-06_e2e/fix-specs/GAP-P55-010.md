# Fix Spec: GAP-P55-010 — Adverse party search broken (partial name tokens fail)

## Problem
Searching for "BHP" or "Road" returns 0 results despite "BHP Minerals SA (Pty) Ltd" and "Road Accident Fund" existing. Searching "Minerals" works. The search is unreliable for short or leading tokens of a name.

## Root Cause (confirmed)
- Repository: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/verticals/legal/conflictcheck/AdversePartyRepository.java` line 18-28
- Query: `WHERE public.similarity(lower(name), lower(:name)) > :threshold`
- Service: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/verticals/legal/conflictcheck/AdversePartyService.java` line 333
  - `FUZZY_THRESHOLD = 0.3`

The issue is that `pg_trgm similarity()` compares the **entire** search term against the **entire** name. Short tokens like "BHP" produce low trigram overlap against a long name like "BHP Minerals SA (Pty) Ltd" because the denominator includes all trigrams in both strings. The similarity score for "BHP" vs "BHP Minerals SA (Pty) Ltd" is likely around 0.15-0.25 (below the 0.3 threshold).

"Minerals" works because it's a longer token with more trigram overlap.

This is not an issue with the threshold alone -- the `similarity()` function is the wrong tool for prefix/substring matching. The correct approach is to combine `ILIKE` for prefix/substring matches with `similarity()` for fuzzy matches.

## Fix
Add `ILIKE` prefix/substring matching as a primary search path, with `similarity()` as a secondary fuzzy fallback.

1. In `AdversePartyRepository.java`, add a new query method:
   ```java
   @Query(
       value = "SELECT * FROM adverse_parties"
               + " WHERE lower(name) LIKE lower(:pattern)"
               + " ORDER BY name ASC"
               + " LIMIT :maxResults",
       nativeQuery = true)
   List<AdverseParty> findByNameContaining(
       @Param("pattern") String pattern,
       @Param("maxResults") int maxResults);
   ```

2. In `AdversePartyService.list()` (line 329-367), change the search logic to first try ILIKE, then supplement with fuzzy:
   ```java
   if (search != null && !search.isBlank()) {
       int maxResults = pageable.isPaged() ? pageable.getPageSize() * 3 : 200;
       
       // Primary: substring match (handles "BHP", "Road", etc.)
       var substringMatches = adversePartyRepository.findByNameContaining(
           "%" + search + "%", maxResults);
       
       // Secondary: fuzzy match (handles typos like "Roadd Accident")
       var fuzzyMatches = adversePartyRepository.findBySimilarName(
           search, FUZZY_THRESHOLD, maxResults);
       
       // Tertiary: alias matches
       var aliasMatches = adversePartyRepository.findByAliasContaining(
           search, FUZZY_THRESHOLD, maxResults);
       
       // Merge and deduplicate, preserving substring-match priority
       var seen = new LinkedHashSet<UUID>();
       var merged = Stream.of(substringMatches.stream(), fuzzyMatches.stream(), aliasMatches.stream())
           .flatMap(Function.identity())
           .filter(ap -> seen.add(ap.getId()))
           .filter(ap -> partyType == null || ap.getPartyType().equals(partyType))
           .toList();
       // ... rest of pagination logic unchanged
   }
   ```
