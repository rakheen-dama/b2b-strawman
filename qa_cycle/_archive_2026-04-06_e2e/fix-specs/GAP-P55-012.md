# Fix Spec: GAP-P55-012 — Tariff Schedules page crashes (data shape mismatch)

## Problem
The Tariffs page (`/legal/tariffs`) crashes with two errors:
1. Server-side: `TypeError: Cannot read properties of undefined (reading 'totalElements')` in `page.tsx` line 31
2. Client-side: `TypeError: Cannot read properties of undefined (reading 'length')` in `TariffBrowserClient`

The backend `GET /api/tariff-schedules` returns a **plain array** (`List<ScheduleResponse>`) but the frontend expects a **paginated response** (`{ content: [...], page: { totalElements } }`).

## Root Cause (confirmed)
- **Backend**: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/verticals/legal/tariff/TariffController.java` line 37
  - `listSchedules()` returns `ResponseEntity<List<ScheduleResponse>>` — a plain JSON array
- **Frontend**: `frontend/app/(app)/org/[slug]/legal/tariffs/actions.ts` line 26-37
  - `fetchTariffSchedules()` returns `Promise<PaginatedResponse<TariffSchedule>>` — expects `{ content, page }`
- **Frontend**: `frontend/app/(app)/org/[slug]/legal/tariffs/page.tsx` line 31-32
  - Accesses `result.content` and `result.page.totalElements` — both undefined when result is an array

The mismatch: the frontend was coded assuming paginated response, but the backend returns a simple list. This is a clear contract mismatch between frontend and backend.

## Fix
**Option A (recommended — fix frontend to match backend):** The tariff schedules are a small dataset (typically < 10 schedules per tenant), so pagination is unnecessary. Fix the frontend to accept an array.

1. In `frontend/app/(app)/org/[slug]/legal/tariffs/actions.ts`, change `fetchTariffSchedules()`:
   ```tsx
   export async function fetchTariffSchedules(): Promise<TariffSchedule[]> {
     return api.get<TariffSchedule[]>("/api/tariff-schedules");
   }
   ```
   Remove unused `page`/`size` params.

2. In `frontend/app/(app)/org/[slug]/legal/tariffs/page.tsx`, change data fetching:
   ```tsx
   try {
     const result = await fetchTariffSchedules();
     initialSchedules = result ?? [];
     initialTotal = initialSchedules.length;
   } catch (error) {
     console.error("Failed to fetch tariff schedules:", error);
   }
   ```

3. In `frontend/app/(app)/org/[slug]/legal/tariffs/tariff-browser-client.tsx`, change `refetch()`:
   ```tsx
   const refetch = useCallback(() => {
     startTransition(async () => {
       try {
         const result = await fetchTariffSchedules();
         setSchedules(result ?? []);
         setTotal(result?.length ?? 0);
       } catch (err) {
         console.error("Failed to refetch tariff schedules:", err);
       }
     });
   }, []);
   ```

4. Similarly fix `fetchTariffItems()` if `GET /api/tariff-items` also returns a plain array (confirmed: `TariffController.searchItems()` line 83 returns `List<TariffItemResponse>`).
