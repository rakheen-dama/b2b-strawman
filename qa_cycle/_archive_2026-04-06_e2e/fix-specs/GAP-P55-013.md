# Fix Spec: GAP-P55-013 — No manual trigger for court date reminder job

## Problem
`CourtDateReminderJob` only runs on cron schedule (6 AM daily). No endpoint exists to trigger it manually for QA/testing. This prevented verification of reminder notifications during QA cycle.

## Triage
**WONT_FIX** for this bugfix cycle.

Rationale:
- Adding an internal endpoint to trigger a cron job is useful but not essential for production functionality.
- The job logic was verified correct via code review during QA.
- GAP-P55-002 (expired status detection) addresses the most important consequence of the job not running.
- Creating a new internal endpoint, wiring it through filters, and adding tests exceeds the scope of a quick bugfix.

## Scope
Backend
Migration needed: no

## Estimated Effort
M (30 min - 2 hr) — if implemented, would involve:
- Adding `POST /internal/jobs/court-date-reminders/trigger` endpoint
- Calling `CourtDateReminderJob.execute()` manually
- Adding API key auth for the internal endpoint
