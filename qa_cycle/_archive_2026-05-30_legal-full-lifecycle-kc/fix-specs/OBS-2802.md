# Fix Spec: OBS-2802 — Billing wizard discoverCustomers() returns 0 customers

## Problem

Billing wizard Step 2 shows "No customers with unbilled work found for this period." with 0 customers and R 0.00 total. Sipho Dlamini has 4h billable time and an approved R 1,250 disbursement, but the wizard excludes him.

## Root Cause

`BillingRunSelectionService.discoverCustomers()` (line 533-535 in `backend/src/main/java/.../billingrun/BillingRunSelectionService.java`) filters on `lifecycle_status = 'ACTIVE'`:

```sql
WHERE c.lifecycle_status = 'ACTIVE'
```

Sipho's `lifecycle_status` is `ONBOARDING` because OBS-2801 (the "Activate Customer" button is a no-op) prevents the lifecycle transition. The backend billing query is correct by design — only ACTIVE customers should be billable.

## Triage Decision: CASCADING — no fix needed

This is a **pure cascading consequence** of OBS-2801. The billing wizard's `lifecycle_status = 'ACTIVE'` filter is **intentionally correct** — firms should not bill customers who are still being onboarded. The backend code does not need changes.

Once OBS-2801 is fixed and Sipho is activated to ACTIVE status, `discoverCustomers()` will return Sipho with his unbilled work (4h time + R 1,250 disbursement) as expected.
