# Fix Spec: OBS-5008 — Invoice empty state terminology leak

## Problem

On the global Invoices page (`/org/thornton-associates/invoices`), the empty state reads:
- Heading: "No fee notes yet"
- Description: "Generate fee notes from tracked time or create them manually. You'll need at least one matter with logged time."

For the `accounting-za` vertical, "matter" should be "engagement" and "fee notes" should be "invoices" (consistent with the page title "Invoices", sidebar label "Invoices", and button "+ New Invoice").

"Fee note" is the `legal-za` terminology for Invoice. "Matter" is the `legal-za` terminology for Project/Engagement. These terms are leaking into the accounting vertical because the empty state messages use hardcoded legal terminology instead of terminology placeholders.

Additionally, the status-filtered empty states (lines 206-210) hardcode "fee notes" in the JSX:
```tsx
title={search.status ? "No fee notes found" : t("invoices.list.heading")}
description={search.status ? `No ${search.status.toLowerCase()} fee notes found.` : t("invoices.list.description")}
```

Evidence: OBS-5008 in `qa_cycle/checkpoint-results/day-36-60.md`.

## Root Cause (confirmed)

**Same pattern as OBS-5001 (projects empty state).** The fix is identical in shape.

### Source 1: empty-states.json (lines 46-52)

`frontend/lib/messages/en/empty-states.json` contains:
```json
"invoices": {
  "list": {
    "heading": "No fee notes yet",
    "description": "Generate fee notes from tracked time or create them manually. You'll need at least one matter with logged time.",
    "cta": "Create fee note"
  }
}
```

These use hardcoded `legal-za` terminology ("fee notes", "matter") instead of terminology placeholders.

### Source 2: Invoices page JSX (lines 206-210)

`frontend/app/(app)/org/[slug]/invoices/page.tsx` passes raw message strings and hardcoded "fee notes" text directly to `<EmptyState>` without wrapping them in `<TerminologyText>`.

## Fix

### Step 1: Update empty-states.json

In `frontend/lib/messages/en/empty-states.json`, update the `invoices.list` messages to use terminology placeholders:

```json
"invoices": {
  "list": {
    "heading": "No {invoices} yet",
    "description": "Generate {invoices} from tracked time or create them manually. You'll need at least one {project} with logged time.",
    "cta": "Create {invoice}"
  }
}
```

This uses `{invoices}` / `{invoice}` / `{project}` placeholders that `TerminologyText` resolves per vertical:
- `accounting-za`: `{invoices}` -> "invoices", `{project}` -> "engagement" (from terminology map)
- `legal-za`: `{invoices}` -> "fee notes", `{project}` -> "matter" (from terminology map)
- Default: `{invoices}` -> "invoices", `{project}` -> "project"

### Step 2: Wrap empty state props in TerminologyText

In `frontend/app/(app)/org/[slug]/invoices/page.tsx`, change lines 203-213 from:

```tsx
<EmptyState
  icon={Receipt}
  title={search.status ? "No fee notes found" : t("invoices.list.heading")}
  description={
    search.status
      ? `No ${search.status.toLowerCase()} fee notes found.`
      : t("invoices.list.description")
  }
  secondaryLink={{ label: "Read the guide", href: docsLink("/features/invoicing") }}
/>
```

To:

```tsx
<EmptyState
  icon={Receipt}
  title={
    search.status
      ? <TerminologyText template={`No ${search.status.toLowerCase()} {invoices} found`} />
      : <TerminologyText template={t("invoices.list.heading")} />
  }
  description={
    search.status
      ? <TerminologyText template={`No ${search.status.toLowerCase()} {invoices} found.`} />
      : <TerminologyText template={t("invoices.list.description")} />
  }
  secondaryLink={{ label: "Read the guide", href: docsLink("/features/invoicing") }}
/>
```

### Step 3: Add TerminologyText import

Add `import { TerminologyText } from "@/components/terminology-text";` to the invoices page. The page already imports `TerminologyHeading` from `@/components/terminology-heading`.

**Note:** `TerminologyText` is a `"use client"` component being rendered inside a Server Component. This is valid because it's passed as a `ReactNode` prop (JSX), not a function/component reference. The `EmptyState` component accepts `React.ReactNode` for both `title` and `description`.

### Step 4: Verify terminology map coverage

Check that the `accounting-za` terminology map in `frontend/lib/terminology-map.ts` has entries for `{invoices}`, `{invoice}`, `{Invoices}`, `{Invoice}`. Currently, the `accounting-za` map does NOT remap Invoice (it stays as "Invoice"), which is correct — only `legal-za` remaps to "Fee Note". The `TerminologyText` component returns the term as-is when no mapping exists, so `{invoices}` becomes "invoices" for accounting-za. This is the desired behavior.

### Step 5: Update tests

Check `frontend/__tests__/` for any tests that assert on the invoice empty state text. If tests render without a `TerminologyProvider`, `TerminologyText` uses the identity function and `{invoices}` becomes "invoices". The rendered text should match "No invoices yet" (default), which is the correct accounting-za behavior. Update any test assertions that expect "fee notes" to expect "invoices" instead.
