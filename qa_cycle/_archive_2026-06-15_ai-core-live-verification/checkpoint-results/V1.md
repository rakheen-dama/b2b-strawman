### Caveat on log proof obligation (for orchestrator)
The turn instructions asked for an Anthropic HTTP-200 + token-count line in `.svc/logs/backend.log`. `validateKey` deliberately uses `BodyHandlers.discarding()` and only logs at **DEBUG** on failure — a *successful* validateKey emits **no INFO/stdout line and no token counts** (it requests `max_tokens:1` and discards the body). So the backend stdout log cannot carry this proof for the connection test specifically; the live-vs-stub proof for V1 is the **latency + `success:true` audit event + anthropic slug routing** triad above.
