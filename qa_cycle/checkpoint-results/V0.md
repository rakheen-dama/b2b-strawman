## Notes / non-defects

- **Session-handoff caveat (testing artefact, not a product defect):** Keycloak SSO keeps the prior actor authenticated, so following an invite registration link while another user's session is live yields a KC "You are already authenticated as different user" page. Resolved during testing by force-logging-out the prior user's KC sessions via the admin API and closing the Playwright browser context (drops the httpOnly gateway-BFF cookie). In real use each invitee opens the link in their own clean browser, so this does not occur. Worth noting for later stages that switch actors (V10 Pieter).
- The platform-admin **Approve** confirmation dialog opens correctly (verified: overlay + `[role=alertdialog]` with `data-state=open` injected into DOM). An early apparent "dialog not opening" was a Playwright click/snapshot race + an accidental double-click toggling it shut — not a product bug.
