### 36.2 Create invoice from unbilled time entries
- **Result**: FAIL (BLOCKER)
- **Evidence**: Clicking "New Invoice" on the customer Invoices tab triggers a prerequisite check server action (`checkPrerequisitesAction("INVOICE_GENERATION", "CUSTOMER", customerId)`) that returns **HTTP 403** with error: *"This feature is not enabled for your organization. An admin can enable it in Settings > Features."* The dialog DOES open (the catch block falls through), displaying a "Generate Invoice" dialog with From Date, To Date, and Currency (ZAR) fields. However, clicking "Fetch Unbilled Time" does NOT advance to step 2 -- the dialog stays stuck on step 1 with no visible error message. The server action returns HTTP 200 but the unbilled time data does not populate, likely because the capabilities check fails silently (owner capabilities not resolving correctly through the Keycloak BFF session).
- **Root causes identified**:
  1. **GAP-D36-01 (HIGH)**: Backend prerequisite check API returns 403 "Module not enabled" for `INVOICE_GENERATION` context. The `accounting-za` vertical profile does not auto-enable the invoicing module. Settings > Features page shows only 3 toggles (Automation Rule Builder, Bulk Billing Runs, Resource Planning) -- none labeled "Invoicing".
  2. **GAP-D36-02 (HIGH)**: "Fetch Unbilled Time" button click does not advance the dialog from step 1 to step 2. The `fetchUnbilledTime` server action completes (200 OK in frontend logs) but the data does not populate in the dialog. Possible cause: the owner role capabilities are not resolving correctly, causing the capability guard (`!caps.isAdmin && !caps.isOwner`) to return `{ success: false }`, which the dialog handles by silently staying on step 1 with no error message shown to the user.
