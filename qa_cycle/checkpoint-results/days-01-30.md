## New Gaps

- **GAP-L-05** (MED, Dev): Activity feed shows task title as literal string "unknown" for task-assignment events. On Sipho's matter overview Recent Activity panel, one entry reads: "Bob Ndlovu assigned task 'unknown'". Other activity entries (logged-time, claimed, created-document) show task titles correctly. Root cause likely an event payload that captures task ID but not title at the point where "task.assigned" is written, while other events look up title at write-time. MED because it impacts audit trail readability (imagine 6 months of "assigned task 'unknown'"); the raw event data is probably fine (task_id persisted). One-line fix likely — resolve title at event-emit. Repro: create a matter, add a project member, open task detail, change assignee → watch activity feed.

## Observations (non-gap)

- **OBS-L-07** (LOW, Product): Day 2.7 scenario expects `checklist 100% → customer auto-transitions to ACTIVE`, but current implementation also requires Tax Number. Both the UI ("Blocking activation" card) and the backend enforce this correctly; just the scenario script doesn't mention tax. Either the scenario or the activation rule should be updated to match expectations. Tax Number requirement is ZA-best-practice (need a SARS tax number to invoice on a FICA-compliant activation), so implementation is probably right — update the scenario.

- **OBS-L-08** (LOW, QA-infra): Radix Select components (e.g., Create Client → Type dropdown) don't accept JavaScript `setter.call(el, val)` on their hidden `<select>` without a real pointer event. Playwright's `browser_fill_form` + `selectOption` works correctly; direct JS doesn't persist the intended value (gets saved as default INDIVIDUAL in the Moroka case — required re-create). QA-tooling limitation, not a product bug. When possible, use Playwright's native label-based fill over JS setter.

- **OBS-L-09** (LOW, Product): Beneficial-owners are captured as a single uploaded declaration PDF in the FICA trust checklist, not as a structured list of individual beneficial owners with names/IDs/percentages. Per FIC Act s21A(2)(b), a declaration listing the BOs is adequate — but a structured list would enable richer queries (e.g., "all matters where John Moroka is a BO"). Product consideration only.

- **OBS-L-10** (LOW, Product): On Lerato's RAF matter, the system permitted matter creation while the customer was in ONBOARDING lifecycle state (not yet ACTIVE). MEMORY.md indicates "CustomerLifecycleGuard blocks CREATE_PROJECT for PROSPECT customers" but may permit ONBOARDING. Works for legal-za (attorneys often open matters during intake and complete FICA in parallel), but worth a product-level review on strict-vs-permissive activation gates.

- **OBS-L-11** (LOW, Product): Adverse Party registry "Party Type" dropdown has 6 options (Natural Person, Company, Trust, Close Corporation, Partnership, Other). No "Government" or "Statutory Body" option — forces RAF (a government statutory entity) to be entered as "COMPANY". Minor taxonomy gap for ZA-legal vertical.

- **OBS-L-12** (LOW, Dev): Small terminology inconsistency — sidebar shows "Action Items" for tasks but matter detail section heading says "Tasks". Breadcrumb uses "Action Items" as the tab label. Scenario expected "Tasks" everywhere, so this is legal-vertical-terminology-trying-to-coexist-with-common-terms.
