# OBS-2103b Implementation Note

**PR**: #1242
**Merge SHA**: `f0845762`
**Merged**: 2026-04-30
**Status**: FIXED (closes both OBS-2103 reopen + OBS-2103b spec)

## Background

OBS-2103 originally fixed a Radix Slot adjacency collision under React 19 (PR
#1239 `a93740d5`) by switching from `<DialogTrigger asChild>` /
`<AlertDialogTrigger asChild>` to manual `React.cloneElement(children, {
onClick })` injection.

QA verified the visibility half passed (both Edit + Archive render
side-by-side on Sipho and Moroka), but the click half FAILED:

- Sipho: Archive AlertDialog opens; Edit click is a no-op (Edit `<button>`
  fiber `memoizedProps` lacks `onClick`).
- Moroka: polarity flipped — Edit dialog opens; Archive click is a no-op.

QA hypothesis (cycle 17): EditCustomerDialog children prop arrives as a
lazy/RSC element where `children.props` is undefined (object keys =
`$$typeof`, `_payload`, `_init`, `_debugInfo`, `_store`), so `cloneElement`
returns an element with default props only. ArchiveCustomerDialog receives
an already-resolved element so its onClick survives.

## Fix Applied

Rather than chase the cloneElement edge case, applied the QA-suggested
structural fix: **the dialog component owns the trigger button**.

### Files Changed

1. `frontend/components/customers/edit-customer-dialog.tsx`
   - Removed `children: React.ReactNode` prop.
   - Added `triggerLabel: ReactNode`, `triggerVariant?: ButtonVariant`
     (default `"outline"`), `triggerSize?: ButtonSize` (default `"sm"`),
     `triggerClassName?: string`, `triggerIcon?: ReactNode`.
   - Removed `cloneElement` / `isValidElement` imports + the trigger
     synthesis block.
   - Renders the trigger directly:
     ```tsx
     <Button
       type="button"
       variant={triggerVariant}
       size={triggerSize}
       className={triggerClassName}
       onClick={() => setOpen(true)}
     >
       {triggerIcon}
       {triggerLabel}
     </Button>
     ```

2. `frontend/components/customers/archive-customer-dialog.tsx`
   - Same prop refactor (defaults: `triggerVariant="ghost"`,
     `triggerSize="sm"`).
   - Same direct `<Button>` render with `onClick={() => setOpen(true)}`.

3. `frontend/app/(app)/org/[slug]/customers/[id]/page.tsx`
   - Updated the call site:
     ```tsx
     <EditCustomerDialog
       customer={customer}
       slug={slug}
       triggerLabel="Edit"
       triggerVariant="outline"
       triggerSize="sm"
       triggerIcon={<Pencil className="mr-1.5 size-4" />}
     />
     <ArchiveCustomerDialog
       slug={slug}
       customerId={customer.id}
       customerName={customer.name}
       triggerLabel="Archive"
       triggerVariant="ghost"
       triggerSize="sm"
       triggerClassName="text-red-600 hover:bg-red-50 hover:text-red-700 dark:text-red-400 dark:hover:bg-red-950 dark:hover:text-red-300"
       triggerIcon={<Archive className="mr-1.5 size-4" />}
     />
     ```

## Why This Closes Both Bugs Together

- **OBS-2103** (Radix Slot adjacency collision) — gone because there's
  no Slot wrapper at all; the dialog renders a plain `<Button>`.
- **OBS-2103b** (cloneElement onClick strip on lazy/RSC children) — gone
  because there's no `cloneElement` and no consumer-supplied children
  element; the dialog wires `onClick={() => setOpen(true)}` directly on
  its own `<Button>`.

## Out of Scope

- Other dialog components in `frontend/components/customers/` that also
  use the `children` API (`AnonymizeCustomerDialog`, `DataExportDialog`,
  `LinkProjectDialog`, `CreateCustomerDialog`) were left untouched — they
  do not share the adjacent-sibling shape that triggered OBS-2103/2103b
  on the customer detail action row, and their existing tests + usages
  pass `<Button>` children directly without lazy wrappers.
