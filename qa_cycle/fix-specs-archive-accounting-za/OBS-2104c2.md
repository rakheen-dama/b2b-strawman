# Fix Spec: OBS-2104c2 — CherryPickStep setState-in-render warning

**Severity**: bug (per "frontend must run clean" mandate)
**Surface**: Frontend — `cherry-pick-step.tsx`
**Effort**: XS (~30 min)

## Problem
After OBS-2104c shipped (PR #1241), the Bulk Billing wizard step 3 (CherryPickStep) emitted a React `setState in render` warning during the disbursement-row expansion render. Surfaced during cycle 17 QA verification.

## Root Cause (verified)
The component called `loadCustomerData(...)` (which performs `setX(...)`) **inside** the `setExpandedIds` updater callback. React 19's strict-mode + the React-Compiler rules treat any setState inside another setter's updater as a setState-in-render side effect.

## Fix
Pull `loadCustomerData(...)` OUT of the `setExpandedIds` updater. Resolve `isCurrentlyExpanded` synchronously before calling `setExpandedIds`, then call `loadCustomerData(...)` separately if needed.

## Scope
- `frontend/components/billing-runs/cherry-pick-step.tsx` — single-file change.

## Verification
- Cycle 18 QA: open Bulk Billing wizard step 3, expand a customer row with disbursements, confirm zero `setState-in-render` warning in dev console (post-reload to clear stale chunks).
