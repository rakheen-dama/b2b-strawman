# OBS-2105 — Matter detail header layout collapse (CSS-only fix)

**Status**: SPEC_READY
**Severity**: Medium (cosmetic, but every matter detail page)
**Filed**: Day 60 prep, cycle 19 (2026-04-30)
**Branch**: `bugfix_cycle_2026-04-30b`

## Verified root cause

`frontend/app/(app)/org/[slug]/projects/[id]/page.tsx:479` parent flex row:

```tsx
<div className="flex items-start justify-between gap-4">
  <div className="min-w-0 flex-1">                            { /* line 480 — TITLE column */ }
    ...title, badges, description, customer, fields...
  </div>
  <div className="flex shrink-0 flex-wrap items-center gap-2"> { /* line 603 — ACTIONS cluster */ }
    Close Matter / Reopen / Generate SoA / Lifecycle / Generate Document / New Engagement Letter / Save as Template / Edit / Delete
  </div>
</div>
```

The action cluster carries `shrink-0`. With `shrink-0`, the cluster cannot be sized below its preferred (= max-content) width by the parent flex. Because of `flex-wrap` on the cluster, *its children* may wrap, but the **cluster's preferred width is still computed from a single-line layout** (CSS flexbox treats `flex-wrap` as a hint about overflow, not a smaller intrinsic size).

When the matter is `Active` AND the actor has owner+manage capabilities (Thandi on RAF-2026-001), the cluster renders **8 buttons** in one line. Their combined natural width is ≈ 1200 px on a 1600 × 1000 viewport. With `gap-4` parent + the cluster claiming 1200 px, the title's `flex-1 min-w-0` is squeezed to **0 px** (the `min-w-0` correctly removes the implicit `min-content` floor, which is what makes the failure visible — the title wraps letter-by-letter rather than overflowing the row).

Verified via DOM walk on `/org/mathebula-partners/projects/b7e319f7-...` at viewport ≥ 1280 px:
- Title `<div.min-w-0.flex-1>` → computed `width: 0px`, `flexBasis: 0%`, `flexShrink: 1`.
- Actions `<div.flex.shrink-0.flex-wrap.items-center.gap-2>` → `width: ~1200px`, `flexShrink: 0`.

Reference: `qa_cycle/evidence/day-60/matter-overview-after-tasks.png`, `trust-balance-zero.png`, `closure-gates-all-green.png` (dialog open over collapsed header).

## Fix

Single-file CSS-only change in `frontend/app/(app)/org/[slug]/projects/[id]/page.tsx`. Two cooperating edits:

### Edit 1 — Allow the row itself to wrap (line 479)

```tsx
// before
<div className="flex items-start justify-between gap-4">

// after
<div className="flex flex-wrap items-start justify-between gap-4">
```

Adding `flex-wrap` on the parent lets the action cluster drop below the title row when the combined natural width exceeds the available space. Below the wrap point, each child gets full container width, so the title column gets its full width back.

### Edit 2 — Drop `shrink-0` from the action cluster, keep `flex-wrap` (line 603)

```tsx
// before
<div className="flex shrink-0 flex-wrap items-center gap-2">

// after
<div className="flex flex-wrap items-center justify-end gap-2">
```

Reasoning:
- Removing `shrink-0` allows the cluster to participate in the flex shrink calculation. Combined with the parent now being `flex-wrap`, the cluster will either fit on the same row as the title (when there's room) or wrap onto the next row.
- `justify-end` keeps the buttons right-aligned when the cluster sits on its own row (preserves the visual right-rail look).
- The cluster's children are individual `<Button>`s with their own intrinsic widths — they don't get squeezed because each is effectively `shrink-0` via Shadcn's button base classes.

### Edit 3 — Set a sensible minimum width on the title column (line 480)

```tsx
// before
<div className="min-w-0 flex-1">

// after
<div className="min-w-0 flex-1 basis-[280px]">
```

`basis-[280px]` gives the title column a 280 px flex basis (≈ enough room for the matter name + status badge on a single line at the smallest sensible viewport). Combined with `flex-1`, when the row doesn't wrap, the title gets all remaining space; when the row wraps, the title still gets a sane minimum before wrapping.
