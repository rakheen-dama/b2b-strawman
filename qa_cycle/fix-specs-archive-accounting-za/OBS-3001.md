# Fix Spec: OBS-3001 — Mock payment integration not seeded during tenant provisioning

## Problem

Day 30 checkpoint: portal shows "Contact Mathebula & Partners to arrange payment" instead of "Pay Now" CTA on fee note INV-0001 (R 1,437.50). The `invoice.paymentUrl` is null because no PAYMENT integration row exists in the tenant's `org_integrations` table. The mock payment adapter exists and is functional -- it simply was never seeded for the tenant created during the QA cycle.

Evidence from `qa_cycle/checkpoint-results/day-30.md`:
- Checkpoint 30.5: "No Pay Now CTA rendered. Page instead shows: 'Contact Mathebula & Partners to arrange payment'. This occurs because `invoice.paymentUrl` is null."
- DB confirmation: `SELECT * FROM tenant_5039f2d497cf.org_integrations` returns 0 rows.
- Chain: `IntegrationRegistry.resolve(PAYMENT)` -> `NoOpPaymentGateway` -> `CreateSessionResult.notSupported()` -> `PaymentLinkService.generatePaymentLink()` returns without setting `paymentUrl`.

## Root Cause (confirmed)

**`TenantProvisioningService` does not inject or call `MockPaymentIntegrationSeeder`.**

File: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/provisioning/TenantProvisioningService.java`

The `provisionTenant()` method (line 140) calls the following seeders in order after schema creation and migration:
- `fieldPackSeeder.seedPacksForTenant()` (line 176)
- `clausePackSeeder.seedPacksForTenant()` (line 178)
- `compliancePackSeeder.seedPacksForTenant()` (line 179)
- `standardReportPackSeeder.seedForTenant()` (line 180)
- `requestPackSeeder.seedPacksForTenant()` (line 181)
- `ratePackSeeder.seedPacksForTenant()` (line 183)
- `projectTemplatePackSeeder.seedPacksForTenant()` (line 184)
- `schedulePackSeeder.seedPacksForTenant()` (line 185)
- `legalTariffSeeder.seedForTenant()` (line 186)
- `verticalProfileReconciliationSeeder.reconcile()` (line 189)

**Missing**: `mockPaymentIntegrationSeeder.seedForTenant()` -- not called, not even injected.

The `PackReconciliationRunner` (line 129 of `PackReconciliationRunner.java`) DOES call `mockPaymentIntegrationSeeder.seedForTenant()`, but it runs at application startup (`@Order(100)` ApplicationRunner). On a clean-slate QA cycle, no tenant schemas exist at startup, so the runner logs "No tenant schemas found -- skipping pack reconciliation" and exits. By the time the tenant is provisioned (Day 0 onboarding), the runner has already completed.

The seeder lists in `TenantProvisioningService` and `PackReconciliationRunner` should be symmetrical -- the reconciliation runner handles existing tenants on restart, and the provisioning service handles new tenants. The mock payment seeder was added to the runner (GAP-L-64 comment at line 128) but never added to the provisioning service.

## Fix

### Step 1: Inject `MockPaymentIntegrationSeeder` into `TenantProvisioningService`

**File**: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/provisioning/TenantProvisioningService.java`

1. Add import: `import io.b2mash.b2b.b2bstrawman.integration.payment.MockPaymentIntegrationSeeder;`
2. Add field: `private final MockPaymentIntegrationSeeder mockPaymentIntegrationSeeder;`
3. Add constructor parameter and assignment.

### Step 2: Call `mockPaymentIntegrationSeeder.seedForTenant()` in `provisionTenant()`

Insert the call after `verticalProfileReconciliationSeeder.reconcile()` (line 189) and before `subscriptionService.createSubscription()` (line 190). This mirrors the order in `PackReconciliationRunner` (lines 126-129).

```java
verticalProfileReconciliationSeeder.reconcile(schemaName, clerkOrgId);
// OBS-3001 — seed dev-only mock PSP adapter for legal-za tenants at provision time.
// PackReconciliationRunner covers existing tenants on restart; this covers new tenants.
mockPaymentIntegrationSeeder.seedForTenant(schemaName, clerkOrgId);
subscriptionService.createSubscription(org.getId());
```

### Step 3: No migration or schema changes needed

The `MockPaymentIntegrationSeeder.seedForTenant()` method is:
- **Profile-gated**: no-op unless active profile is `local`, `dev`, `keycloak`, or `test` (line 55-57).
- **Vertical-gated**: no-op unless tenant has `legal-za` vertical profile (line 63-68).
- **Idempotent**: skips if a PAYMENT integration row already exists (line 70-76).
- **Self-contained**: uses `TenantTransactionHelper` to manage its own tenant-scoped transaction (line 58).

No risk to production or non-legal verticals.
