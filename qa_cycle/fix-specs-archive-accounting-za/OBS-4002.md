# Fix Spec: OBS-4002 — Missing engagement templates: Payroll (monthly) and Tax Return -- Trust not in accounting-za pack

## Problem

Day 0 checkpoint 0.41 expected 7 engagement templates in the accounting-za pack but only 5 are present. The scenario file (step 0.41) lists: "Year-End Pack", "Monthly Bookkeeping", "VAT Return (bi-monthly)", "Payroll (monthly)", "Annual Financial Statements", "Tax Return -- Individual", "Tax Return -- Trust". The actual pack at `backend/src/main/resources/project-template-packs/accounting-za.json` contains:

1. Year-End Pack (Annual Financial Statements) -- 7 tasks
2. Monthly Bookkeeping -- 6 tasks
3. Tax Return -- Individual (ITR12) -- 7 tasks
4. Tax Return -- Company (ITR14) -- 7 tasks
5. VAT Return (VAT201) -- 5 tasks

Missing: "Payroll (monthly)" and a Trust-specific template.

**Cascading impact**: Day 16 step 16.1 requires creating an engagement from template "Annual Trust Financial Statements". Without a trust template in the pack, Day 16 cannot proceed as written. This escalates OBS-4002 from LOW to **MEDIUM** -- it blocks the Moroka Family Trust engagement creation at Day 16.

## Root Cause (hypothesis)

The `accounting-za.json` template pack was authored with 5 templates covering the core accounting workflows but did not include payroll or trust-specific templates. This is a seed data gap, not a code bug.

File: `backend/src/main/resources/project-template-packs/accounting-za.json`
