# Fix Spec: OBS-4003 — Logo upload not tested -- no test logo file available

## Problem

Day 0 checkpoint 0.28 was marked PARTIAL because the QA agent could not find a test logo file to upload. The logo upload UI exists and renders correctly ("Upload Logo" button with "PNG, JPG, or SVG. Max 2 MB." hint), but the QA agent did not locate a suitable test fixture.

## Root Cause (hypothesis)

A test logo file DOES exist at `qa_cycle/test-fixtures/mathebula-logo.png`, but it is only 75 bytes -- likely a placeholder or corrupted file, not a valid PNG image. The QA agent either did not know about this file or correctly assessed it as unusable.

## Fix

Create a valid test logo PNG file at `qa_cycle/test-fixtures/thornton-logo.png` that the QA agent can use for the Thornton & Associates scenario. The file should be:
- A valid PNG image (at minimum a 200x200 solid green square matching brand colour #1B5E20)
- Under 2 MB (matching the upload constraint)
- Named to match the scenario firm

The simplest approach: generate a 200x200 green PNG using ImageMagick or a Python script during QA cycle setup.

Alternatively, the QA agent can generate one inline:

```bash
python3 -c "
import struct, zlib
width, height = 200, 200
def create_png(w, h, r, g, b):
    raw = b''
    for y in range(h):
        raw += b'\x00' + bytes([r, g, b]) * w
    compressed = zlib.compress(raw)
    
    def chunk(ctype, data):
        c = ctype + data
        return struct.pack('>I', len(data)) + c + struct.pack('>I', zlib.crc32(c) & 0xffffffff)
    
    ihdr = struct.pack('>IIBBBBB', w, h, 8, 2, 0, 0, 0)
    return b'\x89PNG\r\n\x1a\n' + chunk(b'IHDR', ihdr) + chunk(b'IDAT', compressed) + chunk(b'IEND', b'')

with open('qa_cycle/test-fixtures/thornton-logo.png', 'wb') as f:
    f.write(create_png(width, height, 0x1B, 0x5E, 0x20))
"
```
