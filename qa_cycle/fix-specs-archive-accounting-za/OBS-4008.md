# Fix Spec: OBS-4008 — Budget Alert Escalation rule fails: Jackson null thresholdPercent deserialization

## Problem

The `budget-alert-escalation` rule from `automation-templates/common.json` fails when the automation engine attempts to deserialize its `triggerConfig`. The rule is defined with an empty trigger config (`"triggerConfig": {}`), which means `thresholdPercent` is `null` when Jackson tries to map it to `BudgetThresholdTriggerConfig(int thresholdPercent)`. Jackson cannot assign `null` to a primitive `int` field, causing a deserialization exception.

The `accounting-za-budget-alert` rule (from `accounting-za.json`) works correctly because it specifies `"triggerConfig": { "thresholdPercent": 80 }`.

## Root Cause (hypothesis)

**Root cause 1 — Primitive type in record:**
- File: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/automation/config/BudgetThresholdTriggerConfig.java`, line 3
- Code: `public record BudgetThresholdTriggerConfig(int thresholdPercent) implements TriggerConfig {}`
- The `int` primitive cannot accept `null`. When the `common.json` escalation rule provides `"triggerConfig": {}`, Jackson's `objectMapper.convertValue(raw, BudgetThresholdTriggerConfig.class)` fails because there is no `thresholdPercent` key in the map, resulting in `null` which cannot be assigned to `int`.

**Root cause 2 — Missing threshold in seed data:**
- File: `backend/src/main/resources/automation-templates/common.json`, lines 63-64
- The `budget-alert-escalation` template defines `"triggerConfig": {}` with no `thresholdPercent` value.
- This is a "catch-all" escalation rule — it should fire for any budget threshold event regardless of percentage. But the empty config still gets deserialized through the `BUDGET_THRESHOLD_REACHED` path in `AutomationConfigDeserializer.java` (line 25-26), which maps it to `BudgetThresholdTriggerConfig`.

**Deserialization path:**
- `TriggerConfigMatcher.matches()` calls `configDeserializer.deserializeTriggerConfig(rule.getTriggerType(), rule.getTriggerConfig())`
- `AutomationConfigDeserializer.deserializeTriggerConfig()` for `BUDGET_THRESHOLD_REACHED` does: `objectMapper.convertValue(raw, BudgetThresholdTriggerConfig.class)`
- With `raw = {}`, Jackson tries to construct `BudgetThresholdTriggerConfig(null)` which fails because `int` is not nullable.

**Matcher behavior:**
- `TriggerConfigMatcher.matchesBudgetThreshold()` (line 93-106) checks `consumedPct >= config.thresholdPercent()`.
- If `thresholdPercent` is `0` (default for `Integer` when null), this would mean "match any threshold" — which is correct for the catch-all escalation rule.

## Fix

### Step 1: Change `int` to `Integer` in BudgetThresholdTriggerConfig
In `BudgetThresholdTriggerConfig.java`, change:
```java
public record BudgetThresholdTriggerConfig(int thresholdPercent) implements TriggerConfig {}
```
to:
```java
public record BudgetThresholdTriggerConfig(Integer thresholdPercent) implements TriggerConfig {}
```

### Step 2: Handle null in TriggerConfigMatcher
In `TriggerConfigMatcher.java`, update `matchesBudgetThreshold()` (line 93) to handle null threshold as "match any":
```java
private boolean matchesBudgetThreshold(BudgetThresholdTriggerConfig config, DomainEvent event) {
    if (event.details() == null) {
        return false;
    }
    Object consumedPctObj = event.details().get("consumed_pct");
    if (consumedPctObj == null) {
        return false;
    }
    // null threshold = catch-all (match any budget event)
    if (config.thresholdPercent() == null) {
        return true;
    }
    double consumedPct =
        consumedPctObj instanceof Number n
            ? n.doubleValue()
            : Double.parseDouble(consumedPctObj.toString());
    return consumedPct >= config.thresholdPercent();
}
```

### Step 3: Fix seed data (optional but recommended)
In `backend/src/main/resources/automation-templates/common.json`, change the `budget-alert-escalation` trigger config from `{}` to `{ "thresholdPercent": null }` for explicitness. Or leave as `{}` since Jackson will now handle it correctly.

### Step 4: Add regression test
Add a test in `AutomationEntityTest.java` covering deserialization of `BudgetThresholdTriggerConfig` with null/missing `thresholdPercent` (currently only tests with an explicit value of 90, line 414-415).
