# OBS-404 — Replace "DocTeams" → "Kazi" in user-facing portal strings

- **Severity**: bug (real branding leak — non-negotiable)
- **Owner**: Dev
- **Effort**: XS (single replace across 3 files; one literal each)
- **Scope**: `portal/` only (frontend marketing surface untouched here — see Notes)

## Problem

The product is **Kazi** (company **b2mash**). "DocTeams" is a retired name and **must not appear in any user-facing UI** (project memory: `feedback_product_name_kazi.md`, project `CLAUDE.md`). The portal still ships three user-facing references to "DocTeams":

1. **Portal footer** — `portal/components/portal-footer.tsx:14` — hard-coded
   `<p className="text-xs text-slate-400">Powered by DocTeams</p>`
   Renders on every authenticated portal page.
2. **Portal login page** — `portal/app/login/page.tsx:137` — fallback org name
   `const orgName = branding?.orgName ?? "DocTeams";`
   Visible on `/login` if the public branding fetch is missing or fails (logo `alt`, page title, headings).
3. **Acceptance page** — `portal/app/accept/[token]/acceptance-page.tsx:138` — fallback org name
   `const orgName = pageData?.orgName ?? "DocTeams";`
   Visible on `/accept/[token]` if `getAcceptancePageData` does not return an `orgName`.

QA flagged (1) on Day 4 (`OBS-404`). (2) and (3) are the same bug class discovered by grep during triage — they MUST ship in the same PR per the brand rule.

## Root Cause

Three independent literals were authored before the brand standard was enforced. There is **no** existing centralized brand constant in `portal/lib/` (verified — `portal/lib/` exposes `api-client.ts`, `auth.ts`, `format.ts`, `nav-items.ts`, `terminology-map.ts`, `terminology.tsx`, `types.ts`, `utils.ts` — none defines `BRAND_NAME`).

## Fix Steps

### Step 1 — Introduce a single brand constant
Create `portal/lib/brand.ts`:

```ts
/**
 * Centralized brand constant for the customer portal.
 * Product name is "Kazi"; company is "b2mash".
 * Never use "DocTeams" — it is a retired name (see project CLAUDE.md and
 * `~/.claude/projects/.../feedback_product_name_kazi.md`).
 */
export const BRAND_NAME = "Kazi" as const;
```

### Step 2 — Replace the three literals

**`portal/components/portal-footer.tsx`** (line 14):
```diff
-import { useBranding } from "@/hooks/use-branding";
+import { useBranding } from "@/hooks/use-branding";
+import { BRAND_NAME } from "@/lib/brand";
@@
-        <p className="text-xs text-slate-400">Powered by DocTeams</p>
+        <p className="text-xs text-slate-400">Powered by {BRAND_NAME}</p>
```

**`portal/app/login/page.tsx`** (line 137):
```diff
+import { BRAND_NAME } from "@/lib/brand";
@@
-  const orgName = branding?.orgName ?? "DocTeams";
+  const orgName = branding?.orgName ?? BRAND_NAME;
```

**`portal/app/accept/[token]/acceptance-page.tsx`** (line 138):
```diff
+import { BRAND_NAME } from "@/lib/brand";
@@
-  const orgName = pageData?.orgName ?? "DocTeams";
+  const orgName = pageData?.orgName ?? BRAND_NAME;
```

### Step 3 — Guardrail (prevent regression)
Add a unit test `portal/lib/__tests__/brand.test.ts` that asserts:
- `BRAND_NAME === "Kazi"`
- A grep-equivalent over `portal/app/**/*.tsx`, `portal/components/**/*.tsx`, `portal/lib/**/*.ts` finds **zero** occurrences of the literal `"DocTeams"` (excluding `node_modules`, `.next`, `__tests__`).

Implementation hint (Vitest):
```ts
import { describe, it, expect } from "vitest";
import { readFileSync, readdirSync, statSync } from "fs";
import { join } from "path";
import { BRAND_NAME } from "@/lib/brand";

describe("brand", () => {
  it("exposes Kazi as the canonical brand name", () => {
    expect(BRAND_NAME).toBe("Kazi");
  });

  it("has no DocTeams references in portal source", () => {
    const roots = ["portal/app", "portal/components", "portal/lib"];
    const offenders: string[] = [];
    function walk(p: string) {
      for (const e of readdirSync(p)) {
        const full = join(p, e);
        if (e === "node_modules" || e === ".next" || e === "__tests__") continue;
        const s = statSync(full);
        if (s.isDirectory()) walk(full);
        else if (/\.(tsx?|jsx?)$/.test(e)) {
          const c = readFileSync(full, "utf8");
          if (c.includes("DocTeams")) offenders.push(full);
        }
      }
    }
    // Walk from process.cwd() (vitest runs from /portal)
    roots.map((r) => r.replace(/^portal\//, "")).forEach(walk);
    expect(offenders).toEqual([]);
  });
});
```

(If the test runner's cwd makes the walk awkward, an even simpler shape is to just `expect(readFileSync("./components/portal-footer.tsx", "utf8")).not.toContain("DocTeams")` for each of the three files.)
