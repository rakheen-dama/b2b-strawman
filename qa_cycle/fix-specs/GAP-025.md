# Fix Spec: GAP-025 — Team member list API calls port 8080 instead of 8081 in E2E stack

## Problem
The Team page header shows "3 members" correctly (from the billing API, which uses server-side fetch), but the member list below shows "No members found" with console error `net::ERR_CONNECTION_REFUSED @ http://localhost:8080/api/members`. The E2E backend runs on port 8081 (mapped from Docker internal port 8080), but the client-side `useOrgMembers()` hook fetches directly from `localhost:8080`.

## Root Cause (confirmed)
**File**: `frontend/lib/auth/client/hooks.ts`, lines 12-13:

```typescript
const BACKEND_URL =
  process.env.NEXT_PUBLIC_BACKEND_URL || "http://localhost:8080";
```

The `NEXT_PUBLIC_BACKEND_URL` environment variable is not set in the E2E Docker compose file (`compose/docker-compose.e2e.yml`). Since this is a `NEXT_PUBLIC_` variable, it must be set at **build time** as a Docker build arg (Next.js inlines public env vars during the build). The compose file only sets `BACKEND_URL` (server-side, runtime) but not `NEXT_PUBLIC_BACKEND_URL` (client-side, build-time).

**File**: `compose/docker-compose.e2e.yml`, lines 117-119 (frontend build args):
```yaml
args:
  NEXT_PUBLIC_AUTH_MODE: mock
  NEXT_PUBLIC_MOCK_IDP_URL: http://localhost:8090
```

Missing: `NEXT_PUBLIC_BACKEND_URL: http://localhost:8081`

Additionally, the `frontend/Dockerfile` must declare `ARG NEXT_PUBLIC_BACKEND_URL` for Docker to pass it through (per MEMORY.md lesson about Docker silently ignoring undeclared ARGs).

## Fix

### Step 1: Add `NEXT_PUBLIC_BACKEND_URL` build arg to Docker compose
**File**: `compose/docker-compose.e2e.yml`

Add the build arg to the frontend service:
```yaml
frontend:
  build:
    context: ../frontend
    dockerfile: Dockerfile
    args:
      NEXT_PUBLIC_AUTH_MODE: mock
      NEXT_PUBLIC_MOCK_IDP_URL: http://localhost:8090
      NEXT_PUBLIC_BACKEND_URL: http://localhost:8081    # <-- ADD THIS
```

### Step 2: Declare the ARG in the frontend Dockerfile
**File**: `frontend/Dockerfile`

Add `ARG NEXT_PUBLIC_BACKEND_URL` before the build step (alongside the existing `ARG NEXT_PUBLIC_AUTH_MODE` and `ARG NEXT_PUBLIC_MOCK_IDP_URL`). This ensures Docker passes the value into the build context so Next.js can inline it.

Find the existing `ARG NEXT_PUBLIC_AUTH_MODE` line and add:
```dockerfile
ARG NEXT_PUBLIC_BACKEND_URL
```

## Scope
Docker / Frontend config.

Files to modify:
- `compose/docker-compose.e2e.yml` — add `NEXT_PUBLIC_BACKEND_URL` build arg

Files to create: none

Migration needed: no

**Important**: After applying this fix, the E2E stack must be rebuilt (`bash compose/scripts/e2e-rebuild.sh frontend`) since the change is a build-time variable.
