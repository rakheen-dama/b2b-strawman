# Fix Spec: GAP-027 — Customer pages SSR crash after ONBOARDING lifecycle transition

## Problem
After transitioning any customer from PROSPECT to ONBOARDING, all customer-related pages (list and detail) crash with `TypeError: Cannot read properties of null` during server-side rendering. The backend API continues to work correctly. The page rendered correctly once immediately after the transition (client-side navigation) but crashes on all subsequent SSR loads. This blocks all remaining QA checkpoints from Day 1.4 onward.

## Root Cause (hypothesis)
The customer detail page (`frontend/app/(app)/org/[slug]/customers/[id]/page.tsx`) makes multiple parallel API calls during SSR. Several of the data-access paths have insufficient null guards:

1. **Primary suspect — `customerReadiness.requiredFields` null access (line ~329-339)**: The `fetchCustomerReadiness()` call returns `CustomerReadiness` which has `requiredFields: RequiredFieldStatus`. If the backend returns this as `null` (e.g., for a newly ONBOARDING customer with no required fields configured), the code accesses `customerReadiness.requiredFields.total` without a null guard. The try/catch at lines 274-287 catches the *fetch* error but not a null-dereference in the *rendering* code that uses the fetched data.

2. **Secondary suspect — `customerReadiness.requiredFields.fields` mapping (line ~358)**: `customerReadiness.requiredFields.fields.map(...)` will throw if `fields` is null.

3. **List page cascading crash**: The customer list page re-renders via `revalidatePath('/org/${slug}/customers')` after the transition. If the SSR for the list page triggers a prefetch or if the list page itself has a null dereference (e.g., `completenessScores[c.id]` returning undefined and then accessing `.percentage` on it, though this appears guarded), it would also crash.

4. **Hydration mismatch (React error #418)**: The SSR crash produces partial/broken HTML. When the client tries to hydrate, the server HTML doesn't match, triggering the hydration error. This is a symptom, not the root cause.

## Fix
Add defensive null guards to the customer detail page rendering code. This is a **belt-and-suspenders** fix since the exact crash line is hard to determine without live reproduction.

### Step 1: Guard `customerReadiness.requiredFields` access in customer detail page
**File**: `frontend/app/(app)/org/[slug]/customers/[id]/page.tsx`

In the setup steps mapping (~line 309-339), wrap the `requiredFields` access:

```typescript
// Before:
customerReadiness.requiredFields.total === 0

// After:
(customerReadiness.requiredFields?.total ?? 0) === 0
```

Apply the same pattern to all `requiredFields` accesses:
- `customerReadiness.requiredFields.filled` → `customerReadiness.requiredFields?.filled ?? 0`
- `customerReadiness.requiredFields.total` → `customerReadiness.requiredFields?.total ?? 0`
- `customerReadiness.requiredFields.fields` → `customerReadiness.requiredFields?.fields ?? []`

### Step 2: Guard `contextGroups` construction (~line 351-365)
```typescript
// Before:
customerReadiness && customerReadiness.requiredFields.fields.length > 0

// After:
customerReadiness && customerReadiness.requiredFields?.fields && customerReadiness.requiredFields.fields.length > 0
```

### Step 3: Guard `checklistProgress` access (~line 386-389)
Ensure optional chaining on `customerReadiness?.checklistProgress`:
```typescript
// Before (already has optional chaining but verify):
customerReadiness?.checklistProgress?.completed === customerReadiness?.checklistProgress?.total

// Verify this is already safe — if so, no change needed
```

### Step 4: Add error boundary at the customer page level
Create `frontend/app/(app)/org/[slug]/customers/error.tsx` and `frontend/app/(app)/org/[slug]/customers/[id]/error.tsx` — these catch SSR errors and show a recoverable error page instead of crashing the entire application.

```typescript
"use client";
export default function CustomerError({ error, reset }: { error: Error; reset: () => void }) {
  return (
    <div className="flex flex-col items-center py-24 text-center gap-4">
      <h2 className="font-display text-xl">Something went wrong</h2>
      <p className="text-sm text-slate-600">Unable to load customer data. Please try again.</p>
      <button onClick={reset} className="text-sm text-teal-600 hover:underline">Try again</button>
    </div>
  );
}
```
