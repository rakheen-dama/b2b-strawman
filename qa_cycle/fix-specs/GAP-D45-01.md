# Fix Spec: GAP-D45-01 — Court postponement replaces instead of creating new entry

## Problem
When postponing a court date, the existing entry is updated in-place (new date + POSTPONED status) instead of keeping the original row as POSTPONED and creating a new row with SCHEDULED status and the new date.

## Root Cause (confirmed)
The `CourtCalendarService.postponeCourtDate()` method (lines 240-274 in `CourtCalendarService.java`) updates the existing record:

```java
courtDate.setScheduledDate(request.newDate());  // Overwrites date
courtDate.setStatus("POSTPONED");                // Changes status
courtDate.setOutcome("Postponed: " + request.reason());
var saved = courtDateRepository.save(courtDate);  // Updates in-place
```

It should instead:
1. Keep the old record as POSTPONED (with original date preserved)
2. Create a NEW record with the new date and SCHEDULED status

## Fix

**`backend/src/main/java/io/b2mash/b2b/b2bstrawman/verticals/legal/courtcalendar/CourtCalendarService.java`:**

Replace the `postponeCourtDate` method body:

```java
@Transactional
public CourtDateResponse postponeCourtDate(UUID id, PostponeRequest request) {
  moduleGuard.requireModule(MODULE_ID);

  var original = courtDateRepository.findById(id)
      .orElseThrow(() -> new ResourceNotFoundException("CourtDate", id));

  validateTransition(original.getStatus(), "POSTPONED");

  // 1. Mark original as POSTPONED (keep original date)
  original.setStatus("POSTPONED");
  if (request.reason() != null && !request.reason().isBlank()) {
    original.setOutcome("Postponed: " + request.reason());
  }
  original.setUpdatedAt(Instant.now());
  courtDateRepository.save(original);

  // 2. Create new SCHEDULED entry with the new date
  var newDate = new CourtDate(
      original.getProjectId(),
      original.getDateType(),
      request.newDate(),
      original.getScheduledTime(),
      original.getCourtName(),
      original.getCourtReference(),
      original.getJudgeMagistrate(),
      "Rescheduled from " + original.getScheduledDate(),
      original.getReminderDays()
  );
  newDate.setStatus("SCHEDULED");
  var saved = courtDateRepository.save(newDate);

  // Audit both events
  auditService.log(AuditEventBuilder.builder()
      .eventType("court_date.postponed")
      .entityType("court_date")
      .entityId(original.getId())
      .details(Map.of("new_date", request.newDate().toString(),
                       "reason", request.reason() != null ? request.reason() : "",
                       "new_entry_id", saved.getId().toString()))
      .build());

  return toResponse(saved);  // Return the new SCHEDULED entry
}
```

Note: This requires checking that `CourtDate` has a constructor that accepts all these fields, or using a builder/copy method. Adjust constructor call to match the actual entity API.
