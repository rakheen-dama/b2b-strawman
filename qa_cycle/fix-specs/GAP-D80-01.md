# Fix Spec: GAP-D80-01 — /reports/profitability throws error

## Problem
Navigating to `/reports/profitability` or `/reports` shows "Something went wrong" error page. The working profitability page is at `/profitability`. Observed during Day 80 checkpoint 80.3.

## Root Cause (confirmed)
The `/reports` page exists at `frontend/app/(app)/org/[slug]/reports/page.tsx` and fetches report definitions from the backend API (`/api/report-definitions`). The error is most likely a data fetch failure — the `getReportDefinitions()` call throws, and the page's error handling (line 28) catches it and sets `fetchError = true`, but the page still renders with an empty state "Unable to load reports" — NOT "Something went wrong."

The "Something went wrong" error implies an unhandled exception ABOVE the catch block — likely in the `fetchMyCapabilities()` call (line 13) or during rendering.

However, `/reports/profitability` does NOT exist as a route. There is no `frontend/app/(app)/org/[slug]/reports/profitability/` directory. This URL would 404 (or show Next.js error boundary). The correct URL for the profitability page is `/org/{slug}/profitability`.

The sidebar nav already links to `/profitability` (confirmed in `frontend/lib/nav-items.ts`, line 196). The QA agent may have typed the URL manually or followed a breadcrumb.

## Fix
There is no code bug here. The `/profitability` page works correctly and the sidebar links to it correctly. The `/reports/profitability` path simply doesn't exist and shouldn't — it's a wrong URL.

**Optional enhancement**: Add a redirect from `/reports/profitability` to `/profitability` to prevent confusion if someone types the URL manually.

Create a redirect in `frontend/app/(app)/org/[slug]/reports/profitability/page.tsx`:
```tsx
import { redirect } from "next/navigation";

export default async function ReportsProfitabilityRedirect({ params }: { params: Promise<{ slug: string }> }) {
  const { slug } = await params;
  redirect(`/org/${slug}/profitability`);
}
```

Alternatively, add it to `next.config.ts` as a redirect rule.
