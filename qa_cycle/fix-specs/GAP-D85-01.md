# Fix Spec: GAP-D85-01 — No frontend audit log page

## Problem
No audit log page exists in the frontend. `/audit` and `/settings/audit-log` both return 404. The backend records audit events (Phase 6 infrastructure: `AuditEvent` entity, `AuditService`, query API at `/api/audit-events`) but no frontend page has been built to browse them.

Severity: MED — the demo scenario includes showing audit events at Day 85.

## Root Cause (confirmed)
No audit log page has ever been built. The backend has complete audit infrastructure (Phase 6, Epics 50-53) including entity, repository, service, and query API. But the frontend has zero files for an audit log page -- no route, no components, no data fetching.

## Fix

**WONT_FIX** — Same determination as the previous Legal-ZA cycle (2026-04-13).

Building a full audit log UI page from scratch requires:
1. New route: `frontend/app/(app)/org/[slug]/settings/audit-log/page.tsx`
2. New components: AuditLogTable, AuditLogFilters (actor, entity type, date range, event type)
3. Data fetching: Server action to call `/api/audit-events` with pagination and filters
4. UI: Table with columns (timestamp, actor, event type, entity, details), expandable detail rows
5. Nav item addition to settings sidebar
6. Tests: Page tests, component tests

Estimated effort: 4-8 hours -- exceeds the 2-hour threshold for this QA cycle.

**Demo workaround**: The Activity tab on project detail pages already shows audit-derived events (Phase 6.5 activity feed). The Day 85 demo script should pivot to showcase the Activity tab as proof of the audit trail rather than a dedicated audit log page.
