# Fix Spec: GAP-D85-01 — No audit log UI page in frontend

## Problem
No audit log page exists in the frontend. The backend records audit events (Phase 6 infrastructure), but there is no UI to view or filter them. Navigating to `/settings/audit-log` or `/settings/audit` returns 404. Severity: MED — the demo scenario includes showing audit events to a partner audience at Day 85.

## Root Cause (confirmed)
No audit log page has ever been built. The backend has:
- `AuditEvent` entity (Phase 6, V14 migration)
- `AuditEventRepository`
- `AuditService` (logging)
- Audit query API at `/api/audit-events` (from Phase 6, Epic 53)

But the frontend has zero files related to an audit log page — no page component, no route, no components. Grep for "audit.log", "audit-log", "AuditLog" in `*.tsx` returns zero results.

## Fix
**WONT_FIX** — Building a full audit log UI page from scratch requires:
1. New route: `frontend/app/(app)/org/[slug]/settings/audit-log/page.tsx`
2. New components: AuditLogTable, AuditLogFilters (actor, entity type, date range, event type)
3. Data fetching: Server action to call `/api/audit-events` with pagination and filters
4. UI: Table with columns (timestamp, actor, event type, entity, details), expandable detail rows
5. Tests: Page tests, component tests, integration tests

Estimated effort: 4-8 hours minimum for a production-quality page.

**Alternative for demo**: The activity feed on project detail pages (Activity tab) already shows audit-derived events. The demo can use this existing UI to demonstrate the audit trail, pivoting the Day 85 demo script from "dedicated audit log page" to "activity feed demonstrates full audit trail."

**Recommendation**: Mark WONT_FIX for this cycle. Update the Day 85 demo script to showcase the Activity tab instead. Add audit log UI page to a future phase backlog.
