# GAP-P48-002: No "New Invoice" button on invoices list page

**Severity**: major
**Status**: SPEC_READY
**Estimated effort**: 30 min

## Problem

The `/invoices` list page has no "New Invoice" button. Users must navigate to a specific customer's detail page, go to the Invoices tab, and create an invoice from there. This breaks the expected workflow for power users who want to start from the invoices page.

The backend fully supports creating blank draft invoices (no unbilled time required). The `InvoiceGenerationDialog` component exists and works on customer detail pages but requires a `customerId` prop.

## Root Cause

The invoices list page (`frontend/app/(app)/org/[slug]/invoices/page.tsx`) simply never had a "New Invoice" button added. The `InvoiceGenerationDialog` component requires a specific `customerId`, so it can't be used directly on the list page without a customer selection step.

## Fix

### Add a "Create Invoice" button with customer selection

**File: `frontend/app/(app)/org/[slug]/invoices/page.tsx`**

1. Fetch the customer list alongside invoices (server-side):
   ```ts
   let customers: { id: string; name: string }[] = [];
   try {
     const raw = await api.get<Customer[]>("/api/customers");
     customers = (Array.isArray(raw) ? raw : [])
       .filter(c => c.status === "ACTIVE")
       .map(c => ({ id: c.id, name: c.name }));
   } catch { /* non-fatal */ }
   ```

2. Add a client component `NewInvoiceButton` that:
   - Shows a "New Invoice" button in the page header (next to the title)
   - On click, shows a dropdown/dialog to pick a customer
   - After customer selection, navigates to `/org/{slug}/customers/{customerId}?tab=invoices` (the customer detail invoices tab)
   - OR: renders the `InvoiceGenerationDialog` inline after selection

### Alternative (simpler): Navigate to customer picker

**File: `frontend/app/(app)/org/[slug]/invoices/page.tsx`**

Add a `NewInvoiceDropdown` client component:
- Button labeled "New Invoice" with `<Plus>` icon
- On click: opens a Command/Combobox dialog with customer search
- On select: navigates to `/org/{slug}/customers/{customerId}?tab=invoices`

This is simpler because it reuses the existing `InvoiceGenerationDialog` on the customer page without duplicating it.

### Recommended approach

Create a small client component `CreateInvoiceButton` in `frontend/components/invoices/`:
- Accept `customers: { id: string; name: string }[]` and `slug: string` props
- Render a Button + Popover/Command palette for customer search
- On selection, navigate to the customer's invoices tab

Pass customer list from the server component page.
