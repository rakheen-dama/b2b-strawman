# GAP-P48-011: No close-period UI for retainers (+ GAP-P48-003)

**Severity**: major
**Status**: SPEC_READY
**Estimated effort**: 20 min (investigation + minor fix)

## Problem

The retainer detail page does not show a "Close Period" button, so users cannot close a retainer period and generate the associated invoice. The QA script says "No frontend button to trigger close-period."

## Root Cause Analysis

The close-period UI **already exists**. The code shows:

**`frontend/components/retainers/retainer-detail-actions.tsx`** (line 97-107):
```tsx
{currentPeriod?.readyToClose && (
  <Button variant="outline" size="sm" onClick={() => setClosePeriodOpen(true)}>
    <FileCheck className="mr-1.5 size-4" />
    Close Period
  </Button>
)}
```

The button is conditionally rendered when `currentPeriod?.readyToClose` is true.

**`backend/.../retainer/dto/PeriodSummary.java`** (line 31):
```java
boolean readyToClose = p.getStatus() == PeriodStatus.OPEN && !p.getPeriodEnd().isAfter(LocalDate.now());
```

A period is only `readyToClose` when:
1. Status is `OPEN`
2. `periodEnd` is not after today (i.e., the period has ended or ends today)

**In the E2E scenario**: The retainer periods were created recently (same day as the QA cycle). If the period end date is in the future (e.g., end-of-month), then `readyToClose` is `false` and the button is hidden. This is actually **correct behavior** — you shouldn't close a period that hasn't ended yet.

The QA agent expected to see the button but the period end date hasn't arrived yet.

## Verdict

This is **NOT a bug** — the Close Period button exists and works correctly. It only appears when the period has elapsed. The QA observed this because the retainer's current period had a future end date.

### To verify in E2E

The Close Period button would appear if:
1. A retainer with a short period (e.g., weekly, ending today) is created
2. Or the system date advances past the period end

### Recommendation: WONT_FIX

Mark as WONT_FIX. The feature works as designed. The QA scenario didn't advance calendar time past the period boundary.

**GAP-P48-003** (no retainer-specific invoice flow) is the same issue — the close-period flow generates the invoice. Once the button appears, the full flow works.

## Resolution

No code changes needed. Update status to WONT_FIX with documentation that the button is conditional on period end date.
