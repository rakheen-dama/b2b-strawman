# GAP-P48-013: Invoice send does not trigger email notification

**Severity**: minor
**Status**: SPEC_READY
**Estimated effort**: 45 min (investigation-heavy)

## Problem

When an invoice is sent (status transitions to SENT), no email appears in Mailpit. The invoice status changes correctly but no email notification is generated.

## Root Cause Analysis

The email delivery chain is:

1. `InvoiceService.send()` publishes `InvoiceSentEvent` via Spring's `ApplicationEventPublisher`
2. `InvoiceEmailEventListener.onInvoiceSent()` handles the event (after commit)
3. It loads the invoice, finds an invoice document template, generates a PDF
4. `InvoiceEmailService.sendInvoiceEmail()` resolves the email provider via `IntegrationRegistry`
5. `IntegrationRegistry.resolve(IntegrationDomain.EMAIL, EmailProvider.class)` returns the configured provider

**Possible failure points:**

A. **No invoice document template**: The listener checks `documentTemplateRepository.findByPrimaryEntityTypeAndActiveTrueOrderBySortOrder(TemplateEntityType.INVOICE)`. If no INVOICE template exists in the seed data, it logs a warning and returns without sending. The E2E seed may not include an invoice template.

B. **No email integration configured**: `IntegrationRegistry.resolve()` returns the default adapter. If `IntegrationDomain.EMAIL.getDefaultSlug()` maps to `NoOpEmailProvider`, emails are silently dropped.

C. **No customer email**: `InvoiceEmailService.sendInvoiceEmail()` checks `invoice.getCustomerEmail()` â€” if null/blank, it skips.

D. **SMTP not configured for E2E**: The E2E stack has Mailpit but the backend may not be configured with SMTP settings pointing to Mailpit.

**Most likely cause**: (B) or (D). The E2E stack needs an SMTP email integration configured (host: mailpit, port: 1025) and the integration must be enabled for the tenant. Without this, `IntegrationRegistry` falls back to `NoOpEmailProvider`.

## Fix

### Option A: Configure SMTP in E2E seed (recommended)

**File: `compose/seed/` or E2E seed SQL**

Add an `org_integrations` row for the E2E tenant that enables SMTP email:
```sql
INSERT INTO org_integrations (id, domain, slug, enabled, config)
VALUES (gen_random_uuid(), 'EMAIL', 'smtp', true,
  '{"host":"mailpit","port":1025,"username":"","password":"","fromAddress":"noreply@e2e-test-org.local","fromName":"E2E Test Org"}');
```

### Option B: Ensure an invoice template exists in seed

Check if the template seeder includes an INVOICE type template. If not, add one.

### Verification

1. Create and approve an invoice
2. Send the invoice
3. Check Mailpit (http://localhost:8026) for the email
4. Email should contain the invoice PDF as an attachment

## Investigation Steps (before fixing)

1. Check E2E backend logs for `InvoiceEmailEventListener` log messages:
   - "No invoice document template found" = missing template (fix seed)
   - "Skipping invoice email -- no customer email" = missing email on customer
   - No log at all = event listener not triggered or tenant context issue
2. Check if `SmtpEmailProvider` is configured in the E2E profile
3. Check if an INVOICE template exists in the tenant schema
