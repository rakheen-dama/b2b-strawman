# GAP-P48-015: Customer detail shows "0 projects" despite linked projects

**Severity**: major
**Status**: SPEC_READY
**Estimated effort**: 45 min

## Problem

The customer detail page shows "0 projects" in the header text and "No linked projects" in the Projects tab, even when projects are linked to the customer via `customerId` on the Project entity.

## Root Cause Analysis

The customer detail page at `frontend/app/(app)/org/[slug]/customers/[id]/page.tsx` fetches linked projects via:

```ts
linkedProjects = await api.get<Project[]>(`/api/customers/${id}/projects`);
```

This calls the `CustomerController` endpoint `/api/customers/{id}/projects` which uses `CustomerProjectService.getLinkedProjects()` to query the `customer_projects` join table.

The issue is that projects created with `customerId` set on the `Project` entity are linked via `Project.customerId`, but the customer detail page queries the `customer_projects` join table. These are **two different linkage mechanisms**:

1. `Project.customerId` — direct FK on the project entity (set when creating a project "for" a customer)
2. `customer_projects` — many-to-many join table (used by the "Link Project" dialog)

Projects created via the API with `customerId` in the request body get the FK set on `Project.customerId`, but they are NOT inserted into the `customer_projects` join table. The customer detail page only queries the join table.

## Fix

### Backend: Include projects linked via `Project.customerId`

**File: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/customer/CustomerProjectService.java`**

In the `getLinkedProjects(customerId)` method, merge results from both:
1. `customerProjectRepository.findByCustomerId(customerId)` (join table)
2. `projectRepository.findByCustomerId(customerId)` (direct FK)

Deduplicate by project ID before returning.

Alternatively, if `ProjectRepository` doesn't have a `findByCustomerId` method:

**File: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/project/ProjectRepository.java`**

Add: `List<Project> findByCustomerId(UUID customerId);`

Then in the service, union both queries and deduplicate.

### Verification

1. Create a project with `customerId` set (via API POST `/api/projects` with `customerId` field)
2. Navigate to the customer detail page
3. Header should show "N projects" (correct count)
4. Projects tab should list the project with name, status, due date
5. "Link Project" dialog should still work independently
