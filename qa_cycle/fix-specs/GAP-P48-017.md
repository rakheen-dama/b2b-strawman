# GAP-P48-017: Budget dialog may close without saving

**Severity**: minor
**Status**: SPEC_READY
**Estimated effort**: 15 min

## Problem

The "Configure budget" dialog closes without saving the budget. The API PUT works correctly when called directly. The QA agent described: "UI dialog closed without saving — possible race condition, but API works."

## Root Cause Analysis

The `BudgetConfigDialog` component at `frontend/components/budget/budget-config-dialog.tsx` uses React 19's form action pattern:

```tsx
<form action={handleSubmit} className="space-y-4">
```

The `handleSubmit` function is an async function that calls `upsertBudget()` server action. The issue is the form `action` attribute with an async handler.

In React 19 / Next.js 16, using `action={handleSubmit}` on a form can cause the dialog to close if the `Dialog`'s `onOpenChange` is triggered by the form submission event before the async action completes. The `DialogTrigger` uses `asChild` which means the trigger button controls the open state.

The specific race condition:
1. User clicks "Save Budget" (submit button)
2. Form action starts (async)
3. The dialog detects the form submission as a user interaction and may close
4. The async action completes but `setOpen(false)` is already done

**More likely cause**: The Radix Dialog's internal focus management or the form's default submit behavior causes a state change that closes the dialog before the async handler finishes.

## Fix

**File: `frontend/components/budget/budget-config-dialog.tsx`**

Change the form from using `action=` to using `onSubmit` with `preventDefault`:

```tsx
<form onSubmit={async (e) => { e.preventDefault(); await handleSubmit(new FormData(e.currentTarget)); }} className="space-y-4">
```

Or change the submit button to `type="button"` and call `handleSubmit` directly:

```tsx
<Button type="button" disabled={isSubmitting} onClick={async () => {
  const form = document.getElementById('budget-form') as HTMLFormElement;
  await handleSubmit(new FormData(form));
}}>
```

**Recommended approach**: Convert the `action=` pattern to `onSubmit` with `preventDefault`. This prevents the default form submission from interfering with the dialog state.

1. Navigate to a project's Budget tab
2. Click "Configure budget" (or "Edit" if budget exists)
3. Fill in hours (e.g., 10) and amount (e.g., 5000)
4. Click "Save Budget"
5. Dialog should show "Saving..." state and close only after success
6. Budget should appear in the Budget tab with correct values
