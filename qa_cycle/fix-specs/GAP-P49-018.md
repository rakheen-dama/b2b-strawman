# Fix Spec: GAP-P49-018 — Invoice/statement table amounts lack currency and date formatting

## Problem

In `loopTable` nodes (invoice line items table, statement invoice history table), amounts render as raw numbers (e.g., "5500.00") without currency symbol, and dates render as raw ISO (e.g., "2026-03-17") instead of human-readable format. The Amount Summary section in the same invoice template correctly uses "R5 500,00" format. This inconsistency within the same document looks unprofessional.

## Root Cause (confirmed)

The `TiptapRenderer.renderLoopTable()` method (line 296) renders cell values with plain `String.valueOf(val)`:

```java
Object val = colKey != null ? row.get(colKey) : null;
String cellText = val != null ? HtmlUtils.htmlEscape(String.valueOf(val)) : "";
```

No `VariableFormatter` or format hints are applied. In contrast, the `variable` node handler (line 131) calls `resolveVariable()` which calls `VariableFormatter.format(current, typeHint)` with type hints from `VariableMetadataRegistry`.

The `loopTable` column definitions in the template JSON only specify `header` and `key` -- no `format` or `type` attribute exists.

## Fix

**Add optional `format` attribute to loopTable columns** and apply `VariableFormatter` in `renderLoopTable()`.

### Part A — Template JSON updates (~15 min):

Add a `"format"` attribute to columns that need formatting:

**`invoice-za.json`** loopTable columns:
```json
{ "header": "Qty", "key": "quantity", "format": "number" },
{ "header": "Unit Price (excl. VAT)", "key": "unitPrice", "format": "currency" },
{ "header": "VAT", "key": "taxAmount", "format": "currency" },
{ "header": "Total (incl. VAT)", "key": "lineTotal", "format": "currency" }
```

**`statement-of-account.json`** loopTable columns:
```json
{ "header": "Date", "key": "issueDate", "format": "date" },
{ "header": "Due Date", "key": "dueDate", "format": "date" },
{ "header": "Amount", "key": "total", "format": "currency" },
{ "header": "Running Balance", "key": "runningBalance", "format": "currency" }
```

### Part B — TiptapRenderer change (~20 min):

In `renderLoopTable()`, read the optional `format` attribute from each column definition and pass it to `VariableFormatter.format()`:

```java
for (var col : columns) {
    String colKey = (String) col.get("key");
    String colFormat = (String) col.get("format"); // NEW
    Object val = colKey != null ? row.get(colKey) : null;
    String cellText = val != null
        ? VariableFormatter.format(val, colFormat)  // CHANGED: was String.valueOf(val)
        : "";
    sb.append("<td>").append(cellText).append("</td>");
}
```

Note: `VariableFormatter.format()` already HTML-escapes its output, so remove the `HtmlUtils.htmlEscape()` wrapper.
