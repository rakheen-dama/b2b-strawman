# Fix Spec: GAP-P55-002 — Prescription tracker does not detect expired status

## Problem
A prescription tracker with `prescriptionDate = 2026-01-10` (84 days in the past) shows status "RUNNING" instead of "EXPIRED" or "WARNED". The status field is stored statically in the database and is only set to "RUNNING" at creation time. There is no dynamic status evaluation at read time.

## Root Cause (confirmed)
- Entity: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/verticals/legal/courtcalendar/PrescriptionTracker.java`
  - Constructor sets `this.status = "RUNNING"` (line 78). Status is never re-evaluated.
- Service: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/verticals/legal/courtcalendar/PrescriptionTrackerService.java`
  - The `buildResponse()` method (line 306-325) returns `entity.getStatus()` directly without comparing `prescriptionDate` against `LocalDate.now()`.
  - The `CourtDateReminderJob` (cron, 6 AM daily) may update statuses, but it only runs on schedule and was never triggered during QA.

The status is never dynamically computed at query time. It relies entirely on the batch job which may not have run.

## Fix
Add dynamic status evaluation in the service's response builder. This ensures correct status regardless of whether the reminder job has run.

1. In `PrescriptionTrackerService.java`, modify `buildResponse()` to compute effective status:
   ```java
   private String computeEffectiveStatus(PrescriptionTracker entity) {
       // Terminal states are final
       if ("INTERRUPTED".equals(entity.getStatus()) || "EXPIRED".equals(entity.getStatus())) {
           return entity.getStatus();
       }
       LocalDate today = LocalDate.now();
       if (!entity.getPrescriptionDate().isAfter(today)) {
           return "EXPIRED";
       }
       long daysRemaining = ChronoUnit.DAYS.between(today, entity.getPrescriptionDate());
       if (daysRemaining <= 90) {  // warning threshold — check CourtDateReminderJob for the actual value
           return "WARNED";
       }
       return entity.getStatus(); // RUNNING
   }
   ```
2. In `buildResponse()` (line 306), replace `entity.getStatus()` with `computeEffectiveStatus(entity)`.
3. Check `CourtDateReminderJob` for the warning threshold constant and use the same value.
