# Fix Spec: GAP-P55-009 — Dashboard "Upcoming Court Dates" widget shows error

## Problem
The dashboard "Upcoming Court Dates" widget shows "Unable to load court dates." despite court dates existing and the Court Calendar page loading correctly.

## Root Cause (confirmed)
- Widget: `frontend/components/legal/upcoming-court-dates-widget.tsx` line 46-47
  - Calls `fetchUpcoming()` via SWR
- Action: `frontend/app/(app)/org/[slug]/court-calendar/actions.ts` line 235-237
  - `fetchUpcoming()` calls `api.get<UpcomingResponse>("/api/court-calendar/upcoming")`
- Backend: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/verticals/legal/courtcalendar/CourtCalendarController.java` line 100-104
  - The endpoint is mapped at `@GetMapping("/upcoming")` under `@RequestMapping("/api/court-dates")`
  - **Actual URL**: `/api/court-dates/upcoming`
  - **Frontend calls**: `/api/court-calendar/upcoming` (WRONG PATH)

The frontend action uses the wrong URL path. The backend controller base path is `/api/court-dates` (not `/api/court-calendar`), so the upcoming endpoint is at `/api/court-dates/upcoming`.

## Fix
1. In `frontend/app/(app)/org/[slug]/court-calendar/actions.ts` line 236, change:
   ```tsx
   // Before:
   return api.get<UpcomingResponse>("/api/court-calendar/upcoming");
   // After:
   return api.get<UpcomingResponse>("/api/court-dates/upcoming");
   ```

2. Additionally, the `UpcomingResponse` type (line 230-233) declares `courtDates: CourtDate[]` but the backend returns `UpcomingCourtDateItem` objects with a different shape (has `daysUntil`, no `customerId`, `customerName`, `courtReference`, `judgeMagistrate`, `description`, `outcome`, `reminderDays`, `createdBy`, `createdAt`, `updatedAt`). However, the widget only accesses `cd.id`, `cd.scheduledDate`, `cd.dateType`, `cd.projectName`, and `cd.courtName` — all of which exist on `UpcomingCourtDateItem`. So the type mismatch is cosmetic and the widget will work once the URL is fixed.
