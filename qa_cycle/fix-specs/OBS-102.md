# Fix Spec: OBS-102 — Trust Accounting Settings missing from Settings sidebar

## Problem
The Settings sidebar (rendered at every `/org/{slug}/settings/*` route) has **no entry** for Trust Accounting Settings. The page itself exists at `/org/{slug}/settings/trust-accounting` and works correctly when reached by direct URL, but a real Owner/Admin user navigating Settings has no way to discover it. On Day 1 the QA driver had to type the URL by hand to add a trust account; an end-user encountering this for the first time would assume the firm cannot configure trust accounting at all.

Evidence: `qa_cycle/checkpoint-results/day-01.md` checkpoint 1.5 — "Settings sidebar has no 'Trust Accounts' link." Ditto for the Finance group in `frontend/components/settings/settings-nav-groups.ts`.

## Root Cause (verified)
`frontend/components/settings/settings-nav-groups.ts` defines `SETTINGS_NAV_GROUPS` (the source of truth for the Settings left rail). The `finance` group lists `Rates & Currency`, `Tax`, `Batch Billing`, `Capacity` — but **no Trust Accounting entry**, even though:

- The route `app/(app)/org/[slug]/settings/trust-accounting/page.tsx` exists (315 lines, fully functional).
- `frontend/lib/nav-items.ts` already exposes the page in `SETTINGS_ITEMS` (line 466–472) for the Settings hub card grid and command-palette indexing — but `SETTINGS_ITEMS` is consumed by the hub page, not by `SettingsSidebar`.
- The sidebar component (`components/settings/settings-sidebar.tsx`) iterates `SETTINGS_NAV_GROUPS` only.
- `SettingsSidebar` already supports the `requiredModule` filter (line 43, 83) and the `adminOnly` filter (line 42, 81), so a gated entry will hide cleanly when the org doesn't have the trust module.

## Fix

Add one item to the `finance` group in `frontend/components/settings/settings-nav-groups.ts`:

```ts
{
  id: "finance",
  label: "Finance",
  items: [
    { label: "Rates & Currency", href: "rates" },
    { label: "Tax", href: "tax" },
    {
      label: "Batch Billing",
      href: "batch-billing",
      adminOnly: true,
      requiredModule: "bulk_billing",
    },
    { label: "Capacity", href: "capacity" },
    {
      label: "Trust Accounting",
      href: "trust-accounting",
      adminOnly: true,
      requiredModule: "trust_accounting",
    },
  ],
},
```

Gating reasoning:
- `adminOnly: true` — matches the page-level `caps.isAdmin || caps.isOwner` guard in `app/(app)/org/[slug]/settings/trust-accounting/page.tsx` (line 36). Members would otherwise see the link and hit the "no permission" body.
- `requiredModule: "trust_accounting"` — matches the page-level module guard (line 30). Non-legal verticals (accounting, consulting) won't see the link.

No other code changes needed — the sidebar already filters on both flags.
