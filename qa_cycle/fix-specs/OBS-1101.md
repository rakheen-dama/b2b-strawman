# OBS-1101 — Fix Spec

**Cycle Day**: 11
**Severity**: Cosmetic-but-unprofessional
**Surface**: Backend email — `portal-trust-activity` template

## Symptom

Trust-deposit nudge email body shows raw ISO timestamp
`2026-04-30T10:37:59.248822Z` and raw unformatted amount `50000` instead
of the formatted `30 Apr 2026` and `R 50 000,00` (ZAR) the in-app portal
already renders via `frontend/lib/format.ts` (`formatLocalDate` +
`formatCurrency`).

## Root cause

`PortalEmailNotificationChannel.buildTrustActivityContext(...)` (both the
`TrustTransactionApprovalEvent` and `TrustTransactionRecordedEvent`
overloads) put the raw `BigDecimal amount` and `Instant occurredAt`
straight into the Thymeleaf context, and the template renders them
as-is. There is no pre-format step, and no inline Thymeleaf
`#temporals` / `#numbers` formatting either.

## Fix shape (chosen approach)

**Context-side pre-formatting** — mirrors the OBS-703 `expiresAtFormatted`
pattern in `ProposalSentEmailHandler.java`. Keeps Thymeleaf logic
minimal and lets the handler own locale / currency resolution.

1. Add `amountFormatted` + `occurredAtFormatted` keys to the trust
   context map.
2. Resolve the tenant's `defaultCurrency` from `OrgSettings`, falling
   back to `ZAR` when settings are missing.
3. Hard-code the ZA conventions (`R` symbol, space grouping separator,
   comma decimal separator) instead of trusting
   `NumberFormat.getCurrencyInstance(en-ZA)` — the JDK locale-data
   provider does not reliably resolve `en-ZA` on every distribution
   (test runs reproducibly fell back to `$50,000.00`).
4. Format the timestamp at UTC with `DateTimeFormatter.ofPattern("d MMM yyyy", Locale.ENGLISH)`.
5. Update `templates/email/portal-trust-activity.html` to render the new
   keys with raw-value fallback for safety.

## Files touched

- `backend/src/main/java/io/b2mash/b2b/b2bstrawman/portal/notification/PortalEmailNotificationChannel.java`
- `backend/src/main/resources/templates/email/portal-trust-activity.html`
- `backend/src/test/java/io/b2mash/b2b/b2bstrawman/portal/notification/PortalEmailNotificationChannelIntegrationTest.java` — seed ZAR `OrgSettings` in `@BeforeAll`; assert formatted body in the deposit test (no raw ISO, no raw integer, regex-match `R 50 000,00`).

## Out of scope

- Other email templates (deadline reminder, retainer closed, document
  ready) — not in the OBS-1101 report.
- Plain-text part of the email — Thymeleaf renders both HTML and a
  derived plain-text alternative, both pick up the same context keys.
- A new locale/currency utility — guard rail says "use existing
  patterns or simple inline".
