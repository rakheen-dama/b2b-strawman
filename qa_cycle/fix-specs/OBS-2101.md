# OBS-2101 — LSSA tariff dropdown missing from time-entry dialog

**Source**: Day 21 Phase A (`qa/testplan/demos/legal-za-full-lifecycle-keycloak.md` lines 561-568) — first attempt at scenario step 21.2 ("select tariff activity 'Consultation with client — per 15 minutes' from LSSA dropdown") and 21.3 ("rate auto-populates from tariff").

**Severity**: bug (M effort)

**Status**: OPEN

## Repro

1. Login firm `:3000` as Bob (Admin) at `bob@mathebula-test.local` / `SecureP@ss2`.
2. Navigate to matter RAF-2026-001 (`/org/mathebula-partners/projects/b7e319f7-fd7e-4526-a8b3-b40b1f85b34b`).
3. Click **Tasks** tab → click **Log Time** on any task (e.g. "Initial RAF claim assessment & instructions").

**Observed** — `Log Time` dialog renders only:
- Duration (h / m)
- Date
- Description (optional)
- Billable checkbox
- Yellow warning banner: **"No rate card found for this combination. This time entry will have no billable rate. Set up rates in Project Settings > Rates."**

There is **no tariff/activity dropdown**. There is **no rate auto-populate**. The rate-card warning links to *Project Settings > Rates*, which only supports per-team-member project-level overrides — not the LSSA tariff line items.

**Expected** (per Phase 55 spec + Day 21 scenario)
- A `Tariff Activity` dropdown is shown when matter's vertical profile is `legal-za` (or when a tariff is bound to the matter / firm).
- Selecting an activity (e.g. "Consultation with client — per 15 minutes" from LSSA 2024/2025 schedule) auto-populates the billable rate from the seeded tariff schedule.
- The non-tariff path (free-text time entry) remains as a fallback.

## Evidence

- `qa_cycle/evidence/day-21/log-time-no-tariff.png` — screenshot of dialog showing missing tariff dropdown + the "No rate card found" warning.
- LSSA tariff schedule **is** seeded and visible at `/org/mathebula-partners/legal/tariffs` ("LSSA 2024/2025 High Court Party-and-Party · 19 items"). So the data side of Phase 55 is in place; only the time-entry binding is missing.

## Impact / Why this blocks the scenario

Day 21 → Day 28 dependency chain: scenario 21.4 expects "amount calculated" from tariff; scenario 28.2-28.5 expects bulk-billing run to render fee-note line items as "LSSA tariff line items with activity descriptors". Without tariff binding on time entries, the resulting fee notes will be untyped/zero-rate, breaking the Day 28 cherry-pick + Day 30 PayFast handoff narrative.

Bob CAN log time without tariff (the bare dialog submits successfully), but the entry is non-billable and contributes 0 ZAR to the fee note. So this is functional-blocker for the legal-za vertical's headline workflow, not just cosmetic.

## Suggested fix scope (TBD — for Product/Dev triage)

- Frontend: extend `frontend/components/time/LogTimeDialog.tsx` (or equivalent) to fetch the active tariff schedule for the matter's org/profile (`GET /api/legal/tariffs/active` or similar) and render a `Activity` combobox above Duration. On select, populate `unitRate` and recompute `amount = unitRate * units` (units derive from the tariff's unit type, e.g. per 15min, per hour, per page, per day).
- Backend: ensure `TimeEntry` entity has a `tariff_item_id` FK (nullable). If missing, add a Flyway migration. The `POST /api/projects/{id}/time-entries` endpoint should accept the FK and persist the rate snapshot.
- Vertical gating: only show the dropdown when org's vertical = `legal-za` (other verticals don't have LSSA tariffs).

This is M effort: ~1 frontend dialog change + 1 read endpoint + (likely) 1 schema field. No API contract break (FK is additive).
