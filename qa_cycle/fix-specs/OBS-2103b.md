# Fix Spec: OBS-2103b — Edit/Archive trigger refactor (v2 of OBS-2103)

**Severity**: bug (cosmetic, but blocks Edit/Archive flows for adjacent customers)
**Surface**: Frontend — `EditCustomerDialog`, `ArchiveCustomerDialog`, customer detail page action row
**Effort**: S (~1.5h)

## Problem
PR #1239 (OBS-2103 v1) dropped `asChild` from both dialogs and used `React.cloneElement(children, { onClick: () => setOpen(true) })` to inject the trigger handler. QA verified visibility (both buttons render side-by-side) but **click was broken**: Edit was a no-op on Sipho, Archive was a no-op on Moroka. Only one of the two adjacent cloneElement-injected onClicks survived React commit.

## Root Cause (verified)
Adjacent `cloneElement` calls on consumer-supplied children clobber each other under React 19 reconciliation when the children ship through Server Components or lazy boundaries — `children.props` can be undefined in those paths, so `cloneElement` returns an element with no `onClick`.

## Fix
Refactor both dialogs so the **dialog component owns the trigger button entirely**. Remove the consumer-supplied-children pattern. Add new props:
- `triggerLabel: string`
- `triggerVariant?: ButtonVariant`
- `triggerSize?: ButtonSize`
- `triggerClassName?: string`
- `triggerIcon?: ReactNode`

The customer detail page action row updates its call site to pass `triggerLabel="Edit" triggerVariant="outline"` / `triggerLabel="Archive" triggerVariant="ghost"` instead of wrapping a child Button.

## Scope
- Frontend only.
- `frontend/components/customers/edit-customer-dialog.tsx`
- `frontend/components/customers/archive-customer-dialog.tsx`
- `frontend/app/(app)/org/[slug]/customers/[id]/page.tsx` (call site)
- 3 test files updated to new API.

## Verification
- Re-run cycle 18 verification: Sipho + Moroka both render Edit + Archive; both fire dialogs on click.
- New regression test: `frontend/__tests__/customer-detail-action-row.test.tsx` with 5 cases covering both customers + remount cycle.
