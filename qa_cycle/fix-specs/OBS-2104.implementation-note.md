## What changed

Single backend file: `backend/src/main/java/io/b2mash/b2b/b2bstrawman/billingrun/BillingRunSelectionService.java`

Two SQL changes inside the same service:

### 1. `discoverCustomers()` (the wizard step 2 query)

- **Currency filter on `time_entries`** — was `te.billing_rate_currency = :currency`, now `(te.billing_rate_currency = :currency OR te.billing_rate_currency IS NULL)`. NULL-currency rows arise from time entries logged before a rate card was configured (OBS-2101 cascade — `RateSnapshotService.snapshotRates` skips when no rate card matches).
- **`billing_rate_snapshot` aggregate** — wrapped in `COALESCE(te.billing_rate_snapshot, 0)` so NULL-rate rows do not poison the SUM. They contribute R 0,00, which the fee-note layer will render with a "no rate" inline note (per OBS-2101 WONT_FIX direction).
- **New `LEFT JOIN legal_disbursements ld`** — gated on `billing_status='UNBILLED'` AND `approval_status='APPROVED'` AND `incurred_date BETWEEN :periodFrom AND :periodTo`. The `legal_disbursements` schema has no `currency` column (legal vertical is single-currency ZAR) and no `markup_percent` column, so the billable amount is `amount + COALESCE(vat_amount, 0)` per row.
- **Aggregates updated** — `unbilled_expense_count` now adds `COUNT(DISTINCT ld.id)`; `unbilled_expense_amount` adds the `legal_disbursements` SUM; HAVING extended to `OR COUNT(DISTINCT ld.id) > 0`.

### 2. `createEntrySelections()` time-entry SQL

Mirrors fix #1 — accepts NULL `billing_rate_currency`. Without this, NULL-currency rows would surface in the wizard step-2 totals but disappear from the per-customer entry list, breaking selection.

Note: `legal_disbursements` are NOT inserted into `billing_run_entry_selections` here — that record set is consumed by the existing TIME_ENTRY/EXPENSE selection paths only. The fee-note generation downstream resolves disbursements via the legal-vertical service path.

## Why APPROVED gate on `legal_disbursements`

Spec calls this Path C (a design decision). The QA repro on Day 28 had Sipho's R 1,250 sheriff fee in `approval_status='DRAFT'`. Including DRAFT disbursements in the wizard would let an un-reviewed expense slip onto a fee note. The legal-vertical convention is that disbursements must be approved before they can be billed. Day 28 QA scenario must be amended to include a disbursement-approval step (Day 21.8b: submit + approve).
