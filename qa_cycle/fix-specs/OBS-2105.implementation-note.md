### Why each edit

1. **Line 479 parent — add `flex-wrap`.** Lets the entire action cluster drop below the title row when their combined natural width exceeds container width. Previously the parent could not wrap, so it had to give the title column whatever room was left after the cluster claimed its full natural width — which was 0 px for the matter-active + owner case (8 buttons ≈ 1200 px on a 1600 px viewport).
2. **Line 480 title column — add `basis-[280px]`.** Gives the title a guaranteed 280 px floor on a single-line layout (enough for the matter name + status badge at the smallest sensible viewport). Combined with the existing `flex-1`, it still expands when extra space is available.
3. **Line 603 action cluster — drop `shrink-0`, add `justify-end`.** Removing `shrink-0` lets the cluster participate in the flex shrink calc so it can either fit on the title row (when there's room) or wrap to its own row. `justify-end` keeps the buttons right-aligned when the cluster sits on its own row, preserving the visual right-rail look. The cluster's children (Shadcn `<Button>`s) have their own intrinsic widths, so they don't get squeezed.
