# OBS-2801 Fix Spec — Disbursement lines not rendered in fee note Line Items table

**Gap ID**: OBS-2801
**Severity**: LOW
**Status**: SPEC_READY
**Day**: 28

## Symptom

Fee note detail page renders only TIME lines in the Line Items table. DISBURSEMENT lines (line_type=DISBURSEMENT) exist in the database and are included in the API response, but are silently dropped by the frontend grouping logic. The subtotal correctly includes the disbursement amount (indicating the backend response is correct), but the individual line is invisible.

## Root Cause

Two defects in the frontend, both introduced when DISBURSEMENT support was added to the backend (ADR-247) but the frontend line table was not updated:

### Defect 1: Type union missing DISBURSEMENT

**File**: `frontend/lib/types/invoice.ts`, line 9

```typescript
export type InvoiceLineType = "TIME" | "EXPENSE" | "RETAINER" | "MANUAL" | "TARIFF" | "FIXED_FEE";
```

The backend `InvoiceLineType` enum (`backend/src/main/java/io/b2mash/b2b/b2bstrawman/invoice/InvoiceLineType.java`, line 11) includes `DISBURSEMENT`, but the frontend TypeScript union does not. This means the line's `lineType` is untyped at the TS level and falls through to the `else` branch of the grouping logic.

### Defect 2: groupLinesByType drops DISBURSEMENT group

**File**: `frontend/components/invoices/invoice-line-table.tsx`, lines 17-41

The `groupLinesByType` function correctly creates a group with key `"DISBURSEMENT"` (line 28 — the fallthrough `type` branch). However, the display loop on line 35 only iterates over a hardcoded list:

```typescript
for (const key of ["TIME", "EXPENSE", "TARIFF", "OTHER"]) {
```

Since `"DISBURSEMENT"` is not in this list, the group is created but never rendered. Additionally, `SECTION_LABELS` (lines 9-14) has no entry for `"DISBURSEMENT"`, so there would be no section header label even if the loop included it.

## Fix

Three changes, all in the frontend:

### Change 1: Add DISBURSEMENT to InvoiceLineType

**File**: `frontend/lib/types/invoice.ts`, line 9

```typescript
// Before:
export type InvoiceLineType = "TIME" | "EXPENSE" | "RETAINER" | "MANUAL" | "TARIFF" | "FIXED_FEE";

// After:
export type InvoiceLineType = "TIME" | "EXPENSE" | "DISBURSEMENT" | "RETAINER" | "MANUAL" | "TARIFF" | "FIXED_FEE";
```

### Change 2: Add DISBURSEMENT to SECTION_LABELS and display order

**File**: `frontend/components/invoices/invoice-line-table.tsx`

Add `DISBURSEMENT` entry to `SECTION_LABELS` (line 9):

```typescript
const SECTION_LABELS: Record<string, string> = {
  TIME: "Time Entries",
  EXPENSE: "Expenses",
  DISBURSEMENT: "Disbursements",  // <-- add
  TARIFF: "Tariff Items",
  OTHER: "Other",
};
```

Add `"DISBURSEMENT"` to the display-order array (line 35). Place it after EXPENSE (logical billing grouping: time, expenses, disbursements, tariffs, other):

```typescript
for (const key of ["TIME", "EXPENSE", "DISBURSEMENT", "TARIFF", "OTHER"]) {
```

### Change 3: Map DISBURSEMENT lineType to its own group key (explicit, not fallthrough)

**File**: `frontend/components/invoices/invoice-line-table.tsx`, lines 22-27

The current fallthrough logic works for DISBURSEMENT (maps to its own key), but make it explicit for clarity and to prevent future regressions:

```typescript
// Before:
const groupKey: string =
  type === "TARIFF"
    ? "TARIFF"
    : type === "MANUAL" || type === "RETAINER" || type === "FIXED_FEE"
      ? "OTHER"
      : type;

// After:
const groupKey: string =
  type === "TARIFF"
    ? "TARIFF"
    : type === "DISBURSEMENT"
      ? "DISBURSEMENT"
      : type === "MANUAL" || type === "RETAINER" || type === "FIXED_FEE"
        ? "OTHER"
        : type;
```
