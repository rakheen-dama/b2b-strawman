# Fix Spec: OBS-2801 — "Activate Customer" button on client detail page is a no-op

## Problem

On the customer detail page (`/org/{slug}/customers/{id}`), the "Activate Customer" button renders correctly when the customer is in ONBOARDING status with a completed checklist, but clicking it does nothing. No API call reaches the gateway/backend. The `lifecycle_status` stays at ONBOARDING. No error, no toast, no feedback. The button is a cosmetic no-op.

This blocks the entire Day 28+ flow: billing wizard discovery requires `lifecycle_status = 'ACTIVE'`, which cascades to fee note generation, portal fee note display, payment recording, and matter closure.

## Root Cause (confirmed by code trace)

The customer detail page (`frontend/app/(app)/org/[slug]/customers/[id]/page.tsx`) is a **Server Component** (no `"use client"` directive). It renders the `ClientHeaderCard` component (which is `"use client"`) but **never passes the `onPrimaryAction` prop**:

```tsx
// page.tsx lines 499-519 — onPrimaryAction is NOT passed
<ClientHeaderCard
  customerId={id}
  customerName={customer.name}
  customerStatus={customer.status}
  lifecycleStatus={customer.lifecycleStatus ?? null}
  // ... many props ...
  customer={customer}
  // onPrimaryAction is MISSING
/>
```

The `ClientHeaderCard` component (`frontend/components/customers/client-header-card.tsx`) renders the smart action button with `onClick={onPrimaryAction}` (line 163), but since the prop is `undefined`, the click does nothing.

Furthermore, the page **cannot** pass a function prop across the RSC serialization boundary (Server Component to Client Component). The working infrastructure exists — `TransitionConfirmDialog` and `LifecycleTransitionDropdown` are fully functional client components that correctly call `transitionCustomerLifecycle` server action — but neither is rendered on this page.

The `LifecycleTransitionDropdown` handles the complete flow: prerequisite checks, confirmation dialog, backend API call (`POST /api/customers/{id}/transition`), and page revalidation. It just needs to be wired into the customer detail page.

## Fix

Create a new `"use client"` wrapper component that:
1. Renders `ClientHeaderCard` with the `onPrimaryAction` callback wired
2. Manages the `TransitionConfirmDialog` state (open/close, target status)
3. Handles the prerequisite check flow for ONBOARDING-to-ACTIVE transitions
4. Triggers `router.refresh()` on success to revalidate the server component data

### Step 1: Create `ClientHeaderCardWithLifecycle` wrapper

Create `frontend/components/customers/client-header-card-with-lifecycle.tsx`:

```tsx
"use client";

import { useState } from "react";
import { useRouter } from "next/navigation";
import { ClientHeaderCard, type ClientHeaderCardProps } from "./client-header-card";
import { TransitionConfirmDialog } from "@/components/compliance/TransitionConfirmDialog";
import { PrerequisiteModal } from "@/components/prerequisite/prerequisite-modal";
import { checkPrerequisitesAction } from "@/lib/actions/prerequisite-actions";
import type { LifecycleStatus } from "@/lib/types";
import type { PrerequisiteCheck, PrerequisiteViolation } from "@/components/prerequisite/types";

interface ClientHeaderCardWithLifecycleProps extends ClientHeaderCardProps {
  /** The target lifecycle status for the smart action button */
  targetLifecycleStatus: LifecycleStatus | null;
}

export function ClientHeaderCardWithLifecycle({
  targetLifecycleStatus,
  ...cardProps
}: ClientHeaderCardWithLifecycleProps) {
  const router = useRouter();
  const [dialogOpen, setDialogOpen] = useState(false);
  const [prereqModalOpen, setPrereqModalOpen] = useState(false);
  const [prereqViolations, setPrereqViolations] = useState<PrerequisiteViolation[]>([]);
  const [checkingPrereqs, setCheckingPrereqs] = useState(false);

  async function handlePrimaryAction() {
    if (!targetLifecycleStatus) return;

    // ONBOARDING -> ACTIVE requires prerequisite check first
    if (
      targetLifecycleStatus === "ACTIVE" &&
      cardProps.lifecycleStatus === "ONBOARDING"
    ) {
      setCheckingPrereqs(true);
      try {
        const check: PrerequisiteCheck = await checkPrerequisitesAction(
          "LIFECYCLE_ACTIVATION",
          "CUSTOMER",
          cardProps.customerId
        );
        if (check.passed) {
          setDialogOpen(true);
        } else {
          setPrereqViolations(check.violations);
          setPrereqModalOpen(true);
        }
      } catch {
        // Fall through to confirm dialog on error
        setDialogOpen(true);
      } finally {
        setCheckingPrereqs(false);
      }
    } else {
      setDialogOpen(true);
    }
  }

  function handleSuccess() {
    router.refresh();
  }

  function handlePrerequisiteFailed(check: PrerequisiteCheck) {
    setPrereqViolations(check.violations);
    setPrereqModalOpen(true);
  }

  function handlePrereqResolved() {
    setPrereqModalOpen(false);
    setDialogOpen(true);
  }

  return (
    <>
      <ClientHeaderCard
        {...cardProps}
        onPrimaryAction={targetLifecycleStatus ? handlePrimaryAction : undefined}
      />

      {targetLifecycleStatus && (
        <TransitionConfirmDialog
          open={dialogOpen}
          onOpenChange={setDialogOpen}
          slug={cardProps.slug}
          customerId={cardProps.customerId}
          targetStatus={targetLifecycleStatus}
          onSuccess={handleSuccess}
          onPrerequisiteFailed={handlePrerequisiteFailed}
        />
      )}

      {prereqModalOpen && (
        <PrerequisiteModal
          open={prereqModalOpen}
          onOpenChange={setPrereqModalOpen}
          context="LIFECYCLE_ACTIVATION"
          violations={prereqViolations}
          entityType="CUSTOMER"
          entityId={cardProps.customerId}
          slug={cardProps.slug}
          onResolved={handlePrereqResolved}
        />
      )}
    </>
  );
}
```
