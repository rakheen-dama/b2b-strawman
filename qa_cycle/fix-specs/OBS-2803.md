# Fix Spec: OBS-2803 — Billing Runs page heading says "Invoices" instead of "Fee Notes"

## Problem

The Billing Runs page at `/org/{slug}/invoices/billing-runs` renders a hardcoded `<h1>Invoices</h1>` (line 56 of the page). For the `legal-za` vertical, this should display "Fee Notes" per the terminology map. The sidebar nav correctly says "Fee Notes", but the page heading uses the generic term.

Evidence: QA Day 28 checkpoint 28.1 noted this as a terminology gap.

## Root Cause (confirmed)

In `frontend/app/(app)/org/[slug]/invoices/billing-runs/page.tsx`, line 56:

```tsx
<h1 className="font-display text-3xl text-slate-950 dark:text-slate-50">Invoices</h1>
```

This is a hardcoded string. The parent invoices list page (`/org/{slug}/invoices/page.tsx`) correctly uses `<TerminologyHeading term="Invoices" />` (lines 128, 146), which resolves to "Fee Notes" for `legal-za` via the terminology map.

The terminology map already has the required entry at `frontend/lib/terminology-map.ts` line 67-68:
```ts
Invoice: "Fee Note",
Invoices: "Fee Notes",
```

Additionally, the tab navigation on lines 70-72 has a hardcoded "Invoices" tab label that should also use terminology:

```tsx
<Link ...>Invoices</Link>
```

## Same bug class as OBS-5001/OBS-5008

This is the same terminology placeholder pattern: hardcoded string that should use `TerminologyHeading` or `TerminologyText`. Can be bundled with same-class fixes.

## Fix

### Step 1: Add terminology imports

Add to `frontend/app/(app)/org/[slug]/invoices/billing-runs/page.tsx`:

```tsx
import { TerminologyHeading } from "@/components/terminology-heading";
import { TerminologyText } from "@/components/terminology-text";
```

### Step 2: Replace hardcoded heading (line 56)

**Before:**
```tsx
<h1 className="font-display text-3xl text-slate-950 dark:text-slate-50">Invoices</h1>
```

**After:**
```tsx
<TerminologyHeading term="Invoices" />
```

### Step 3: Replace hardcoded tab label (line 72)

**Before:**
```tsx
<Link
  href={`/org/${slug}/invoices`}
  className="border-b-2 border-transparent px-4 py-2 text-sm font-medium text-slate-500 hover:text-slate-700 dark:text-slate-400 dark:hover:text-slate-300"
>
  Invoices
</Link>
```

**After:**
```tsx
<Link
  href={`/org/${slug}/invoices`}
  className="border-b-2 border-transparent px-4 py-2 text-sm font-medium text-slate-500 hover:text-slate-700 dark:text-slate-400 dark:hover:text-slate-300"
>
  <TerminologyText template="{Invoices}" />
</Link>
```

### Step 4: Replace EmptyState description terminology (line 107)

**Before:**
```tsx
description="Create a billing run to generate invoices for multiple customers at once."
```

**After:**
```tsx
description={<TerminologyText template="Create a billing run to generate {invoices} for multiple {customers} at once." />}
```

Note: `TerminologyHeading` and `TerminologyText` are `"use client"` components rendered inside a Server Component. This is valid — they are passed as ReactNode (JSX), not function/component references. The same pattern is used in the parent invoices page.
