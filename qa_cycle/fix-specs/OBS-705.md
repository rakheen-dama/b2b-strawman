# Fix Spec: OBS-705 — Submit button says "Create Proposal" instead of "Create Engagement Letter"

## Problem
In the `CreateProposalDialog`, the dialog title correctly uses terminology mapping (`<TerminologyText template="New {Proposal}" />` renders "New Engagement Letter" for legal-za), and the description correctly says "Create an engagement letter for a client engagement." However, the submit button at line 596 of `create-proposal-dialog.tsx` is hardcoded as `"Create Proposal"` — it does not use the `TerminologyText` component, so it always reads "Create Proposal" regardless of vertical profile.

## Root Cause (hypothesis)
In `frontend/components/proposals/create-proposal-dialog.tsx`, line 596:
```tsx
{isSubmitting ? "Creating..." : "Create Proposal"}
```
The submit button text is a plain string, not wrapped in `<TerminologyText>`. The dialog title (line 209) and description (line 212) both correctly use `<TerminologyText template="...{Proposal}..." />`, but the submit button was missed.

## Fix
1. In `frontend/components/proposals/create-proposal-dialog.tsx`, line 596, change:
   ```tsx
   {isSubmitting ? "Creating..." : "Create Proposal"}
   ```
   to:
   ```tsx
   {isSubmitting ? "Creating..." : <TerminologyText template="Create {Proposal}" />}
   ```

This ensures the button reads "Create Engagement Letter" in legal-za profile, "Create Proposal" in other profiles. `TerminologyText` is already imported and used elsewhere in this file.
