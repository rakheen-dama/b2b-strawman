# Fix Spec: OBS-707 — Proposal email From address is noreply@docteams.app, not Kazi-branded

## Problem
The proposal notification email sent to portal contacts has `From: noreply@docteams.app`. This is a legacy "DocTeams" branded domain. Per project conventions (CLAUDE.md Memory: "Product is Kazi, company is b2mash"), all user-visible references should use Kazi branding. The Mailpit evidence from Day 7 checkpoint 7.10 shows `From: noreply@docteams.app`.

## Root Cause (hypothesis)
The email sender address is configured centrally in `backend/src/main/resources/application.yml` line 76:
```yaml
docteams:
  email:
    sender-address: ${EMAIL_SENDER_ADDRESS:noreply@docteams.app}
```

This default value `noreply@docteams.app` flows to both `SmtpEmailProvider` (line 34-35) and `SendGridEmailProvider` (line 47), which read `${docteams.email.sender-address}`. All emails sent via these providers use this From address.

Note: `DemoWelcomeEmailService` already uses the correct `noreply@kazi.app` default (line 33), showing this rebranding was started but not completed for the central config.

## Fix
1. In `backend/src/main/resources/application.yml`, line 76, change:
   ```yaml
   sender-address: ${EMAIL_SENDER_ADDRESS:noreply@docteams.app}
   ```
   to:
   ```yaml
   sender-address: ${EMAIL_SENDER_ADDRESS:noreply@kazi.app}
   ```

2. In `backend/src/main/java/io/b2mash/b2b/b2bstrawman/accessrequest/AccessRequestService.java`, line 44, update the fallback default:
   ```java
   @Value("${docteams.email.sender-address:noreply@docteams.app}") String senderAddress,
   ```
   to:
   ```java
   @Value("${docteams.email.sender-address:noreply@kazi.app}") String senderAddress,
   ```

3. In `backend/src/test/resources/application-test.yml`, line 81, update:
   ```yaml
   sender-address: test@docteams.app
   ```
   to:
   ```yaml
   sender-address: test@kazi.app
   ```

4. Update test hardcoded references in:
   - `backend/src/test/java/io/b2mash/b2b/b2bstrawman/integration/email/SendGridEmailProviderTest.java` line 46: `noreply@docteams.app` -> `noreply@kazi.app`
   - `backend/src/test/java/io/b2mash/b2b/b2bstrawman/integration/email/SmtpEmailProviderIntegrationTest.java` lines 27, 93: `test@docteams.app` -> `test@kazi.app`
