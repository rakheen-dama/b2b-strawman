# Fix Spec: OBS-801 — Portal acceptance banner terminology

## Problem

After accepting proposal PROP-0001 on the portal, the success banner reads:
> "Thank you for accepting this proposal. Your project has been set up."

For legal-za orgs, it should say "Your matter has been set up." (and for accounting-za, "Your engagement has been set up.").

## Root Cause (hypothesis)

The message originates from the backend at:
- `backend/src/main/java/io/b2mash/b2b/b2bstrawman/proposal/PortalProposalService.java:148`
  ```java
  "Thank you for accepting this proposal. Your project has been set up."
  ```

This string is hardcoded with "project" and returned in `PortalAcceptResponse.message`. Both portal implementations display it verbatim:

- `portal/app/(authenticated)/proposals/[id]/page.tsx:95` — `setActionMessage({ type: "success", text: result.message })`
- `frontend/app/portal/(authenticated)/proposals/[id]/page.tsx:139` — `setSuccessMessage(result.message)`

The portal has a working terminology system (`useTerminology()` / `t()`) that maps "project" -> "matter" for legal-za, but neither portal page applies `t()` to the acceptance success message.

## Fix

**Approach: Portal-side terminology substitution (both apps)**

Instead of displaying `result.message` verbatim, compose the success message on the portal side using `t()` for the vertical-specific term. The backend message is treated as a fallback.

### File 1: `portal/app/(authenticated)/proposals/[id]/page.tsx`

At line 95, replace:
```tsx
setActionMessage({ type: "success", text: result.message });
```
with:
```tsx
setActionMessage({
  type: "success",
  text: `Thank you for accepting this ${t("proposal")}. Your ${t("project")} has been set up.`,
});
```

This page already imports `useTerminology` (line 13) and destructures `t` (line 51).

### File 2: `frontend/app/portal/(authenticated)/proposals/[id]/page.tsx`

This page does NOT currently use the terminology system. It needs:

1. Import `useTerminology` from the frontend's terminology module (check if one exists, or add the import).
2. At line 139, replace:
   ```tsx
   setSuccessMessage(result.message);
   ```
   with a terminology-aware message, same pattern as File 1.

**Note:** If the frontend portal does not have a `useTerminology` hook wired up, the backend fix below is the fallback. Check whether the frontend portal has a `TerminologyProvider` in its layout before deciding.

### Alternative (backend complement)

If both portals cannot be fixed consistently, fix the backend too:
- `PortalProposalService.java:148` — change "project" to a terminology-resolved term using OrgSettings.terminologyNamespace, or use a generic term like "engagement" that avoids vertical-specific language.
- Simplest backend-only option: change to `"Thank you for accepting this proposal. Your account has been set up."` — using a neutral word that works across all verticals.

## Scope

- **Files changed**: 1-2 portal/frontend files (portal-side fix), or 1 backend file (generic message fix)
- **Risk**: LOW — cosmetic string change only, no logic change
- **Tests**: No new tests needed — this is a UI string. Verify visually during Day 8 retest.
